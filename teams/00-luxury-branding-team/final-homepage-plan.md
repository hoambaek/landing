## 핵심 결론

뮤즈드마레 홈페이지는 7개 섹션으로 구성된 단일 페이지 스크롤 경험으로, "바다의 시간을 기록하는 디지털 아카이브"라는 콘셉트 아래 지상에서 심해로, 다시 수면으로 올라오는 감정 여정을 설계한다. 5개 팀의 산출물 품질은 전반적으로 높으며, 콰이어트 럭셔리와 아카이브 디스커버리 두 축을 대부분 충실히 반영하고 있다. 주요 조율 사항은 (1) 팀 간 컬렉션 깊이 수치 불일치 통일, (2) Ocean Circle 폼 필드 구성 통일, (3) 압력 체험 효과 적용 범위 통일, (4) 카피팀의 "바다에 남길 한 문장" 폼 필드 채택 여부 결정이다. 컬러 토큰은 미확정 상태를 유지하며, 디자인팀 확정 후 전체 적용한다.

## 팀별 산출물 검토 결과

### UX팀 (03) — 승인

가장 체계적이고 포괄적인 산출물이다. 감정 곡선, 섹션별 스크롤 깊이, 전환 상세, 모바일 적응, 접근성(a11y)까지 빠짐없이 기술했다. 브랜드 기준서의 "압박하지 않는다. 사용자가 스스로 발견하게 설계한다"는 원칙을 가장 충실히 구현하고 있다.

**좋은 점**:
- 감정 곡선(Emotional Arc)을 섹션별로 정의하고, 전환율 터치포인트를 감정 축적 로직으로 설계한 점이 브랜드 철학과 정확히 일치한다.
- Ocean Circle CTA를 S6에서 단 한 번만 노출하고, 다른 섹션에 배너/팝업/플로팅 CTA를 금지한 설계는 콰이어트 럭셔리의 핵심이다.
- prefers-reduced-motion 대응이 상세하고, 감소 모션 모드에서도 기능 손실 없음을 명시한 점이 기준을 충족한다.
- 스크린 리더 플로우, 키보드 내비게이션, 포커스 관리까지 접근성 설계가 완결적이다.

**수정 요청**:
- S2→S3 전환에서 hex 값(#3D5A56, #EFDFBB, #1A365D)을 명시하고 있으나, 컬러 토큰이 미확정 상태이므로 "참조용 톤 방향"으로 표기를 변경할 것.
- 압력 체험 효과 적용 범위를 S2~S5로 정의했으나, 디자인팀은 S1~S3으로 정의했다. 아래 충돌 해결 참조.

### 카피라이팅팀 (04) — 승인

브랜드 보이스를 가장 정확하게 구현한 팀이다. "한 문장이 열 문장보다 강하다"는 지침을 문자 그대로 실천했다.

**좋은 점**:
- 톤 일관성 체크리스트를 자체 수행하여 형용사 수식, 감탄부호, 대문자 강조, 판매 언어 전부 통과를 확인한 점이 전문적이다.
- Section 2(Observation) 관찰 일기의 "바다가 숙성을 결정한다. 우리가 아니다."는 브랜드 태도의 핵심을 한 문장으로 요약한다. 확정.
- Section 4(The Maker) 편지의 "이것이 내가 만든 것의 전부다."는 겸손과 자부심의 교차가 정확하다. CTA를 두지 않는 이유("편지에 버튼을 다는 것은 품위를 해친다")도 브랜드 원칙에 부합한다. 확정.
- Section 6(Ocean Circle) "가입이 아니다. 초대다." 두 문장의 힘이 뛰어나다. 확정.
- 테이스팅 리추얼의 시적 안내("소금. 미역. 생굴. 바다에서 온 것은 바다와 함께.")는 지시문이 아닌 서사로서 품위를 유지한다. 확정.

**수정 요청**:
- Ocean Circle 폼 필드에 "바다에 남길 한 문장 (선택)"을 제안했는데, 이 필드는 의례적 장치로서 가치가 있으나, 마케팅팀 및 프론트엔드팀의 폼 설계와 충돌한다. 아래 충돌 해결 참조.

### 마케팅팀 (05) — 조건부 승인

퍼널 설계와 KPI 체계가 체계적이며, PostHog 이벤트 추적 계획까지 상세하다. 경쟁 포지셔닝 매트릭스와 가격 비공개 전략도 브랜드 방향에 부합한다.

**좋은 점**:
- AIDA 매핑을 7개 섹션에 정밀하게 배치한 점이 실용적이다.
- "가격을 공개하지 않는다. 가격이 보이면 상품이 된다."는 원칙이 콰이어트 럭셔리의 핵심을 정확히 이해하고 있다.
- 드롭오프 포인트별 완화 전략이 구체적이다.
- B2B 리드 등급 분류(Hot/Warm/Cold)와 응답 시간 기준이 실무적으로 유용하다.

**럭셔리 기준 위반 및 수정 요청**:

1. **언어 위반 — "감각 충격"(Section 1 핵심 메시지)**: "충격"은 과도한 표현이다. "감각 침잠" 또는 "감각의 고요한 열림"으로 수정할 것. 콰이어트 럭셔리는 충격을 주지 않는다. 서서히 스며든다.

2. **언어 위반 — "소유 욕구 자극"(Section 5 감정 트리거)**: "자극"은 판매 언어에 가깝다. "소유의 조용한 욕구가 형성된다"로 수정할 것.

3. **언어 위반 — "긴급성(조용한)"(Section 6 감정 트리거)**: 긴급성이라는 단어 자체가 콰이어트 럭셔리와 양립 불가하다. 괄호 안에 "조용한"을 붙여도 해소되지 않는다. "놓치고 싶지 않다"도 FOMO 기반 전환 심리이므로 삭제할 것. "이 원 안에 속하고 싶다. 자연스러운 소속 욕구."로 수정할 것.

4. **언어 위반 — "당신의 공간에 바다의 시간을 들이세요"(Section 7 핵심 메시지)**: "들이세요"는 명령형이다. 카피팀의 "바다의 시간을 테이블 위에 놓는 일."이 더 브랜드에 맞다. 카피팀 안으로 통일할 것.

5. **"파트너십 문의"(Section 7 폼 헤더)**: 카피팀은 별도 헤더 없이 문의 폼 라벨만 제시했다. "파트너십 문의"라는 레이블은 기업 브로슈어 톤이다. 삭제하고 카피팀의 "담당자명. 소속. 업종." 라벨 방식으로 통일할 것.

6. **"문의 전달하기"(Section 7 CTA)**: 카피팀의 "문의하다 →"가 더 절제되어 있다. 카피팀 안으로 통일할 것.

7. **Ocean Circle 전환율 12-15%**: UX팀은 5%(도달자 기준)로 설정했다. 마케팅팀의 12-15%는 낙관적이다. 보수적 기준인 UX팀의 5%를 공식 목표로 하되, 마케팅팀의 12-15%를 상한 시나리오로 병기할 것.

### 프론트엔드팀 (01) — 조건부 승인

기술 스펙이 상세하고, 컴포넌트 트리, Supabase 스키마, RLS 정책, 폼 시스템까지 구현 준비가 완결적이다. 성능 예산 관리도 체계적이다.

**좋은 점**:
- WebGL/Three.js 제거 확정 판단이 적절하다. CSS/SVG/Framer Motion/GSAP 조합으로 모든 시각 효과를 구현하는 방향은 성능과 접근성 양쪽에서 안전하다.
- Client vs Server Component 판단 기준을 명확히 정리한 점이 유용하다.
- First Load JS 90KB 목표는 LCP 2.0s 기준 달성에 중요하다.
- Code splitting 전략(dynamic import + SectionSkeleton)이 적절하다.

**수정 요청**:

1. **First Load JS 실측치 우려**: 문서 자체가 "실측치는 ~130KB가 될 수 있음"을 인정하고 있다. 90KB 목표 달성을 위해 Framer Motion LazyMotion + domMax 적용을 필수로 할 것. 130KB는 LCP 2.0s 기준에 위험하다.

2. **컬렉션 깊이 수치 불일치**: 제품 인터페이스에서 `depth_m: 15 | 30 | 50 | 80 | 120`으로 정의했으나, 카피팀은 `20m | 30m | 40m | 50m | 60m`으로, PLAN.md도 60m 기준이다. 아래 충돌 해결 참조.

3. **Ocean Circle 폼 필드 구성**: `name, email, phone, agreeToTerms`로 정의했으나, 카피팀은 `이름, 연락처, 바다에 남길 한 문장(선택)`으로, 마케팅팀은 `이메일, 이름(성), 관심분야(선택)`로 제안했다. 아래 충돌 해결 참조.

4. **Data Archive DepthSelector**: 깊이 선택 수직 슬라이더(15m~120m 전환)를 제안했으나, 이 인터랙션이 "과잉"에 해당할 수 있다. UX팀은 3개 데이터(수온/압력/해류)의 호버 하이라이트만 제안했다. 깊이 선택기는 제거하고, UX팀의 호버/탭 하이라이트 방식으로 단순화할 것.

5. **MembershipForm 성공 메시지**: "바다의 원에 오신 것을 환영합니다."로 구현되어 있으나, 카피팀/UX팀의 "감사합니다. 바다가 답할 것입니다."가 더 브랜드에 맞다. 카피/UX 안으로 통일할 것.

6. **B2B 문의 확인 메시지**: "등록 중 오류가 발생했습니다."는 기술적 표현이다. "잠시 후 다시 시도해주세요."(이미 에러 메시지에 사용)를 기본으로 하되, 성공 메시지는 마케팅팀의 "문의가 전달되었습니다. 3영업일 이내 회신드리겠습니다."가 아니라 UX팀의 "감사합니다. 영업일 기준 2일 내 회신드리겠습니다."로 통일할 것 (2일이 더 빠르고 신뢰감 있음).

### 디자인팀 (02) — 승인

비주얼 시스템의 기반이 탄탄하다. 에디토리얼 미니멀리즘의 원칙("하나의 화면에 하나의 메시지만 존재한다")이 브랜드 기준서와 완전히 일치한다.

**좋은 점**:
- 무드 프로그레션(섹션별 감정 아크 + 온도 감각)이 매우 유용하다. 각 섹션에 "온도" 개념을 부여한 점이 감각적이고, 이 온도 맵이 컬러/텍스처 결정의 기준이 된다.
- 배경 컬러 프로그레션(Dark → Light → Dark → Light → Light → Dark → Light)의 리듬이 스크롤 시 시각적 호흡을 만든다.
- 타이포그래피 체계가 상세하고, 한글 본문 특이사항(양쪽 정렬 금지, line-height 1.75~1.8)이 가독성을 보장한다.
- 이미지 가이드의 "피사체를 중앙에 놓지 않는다" 원칙이 에디토리얼 감각에 맞다.
- 플레이스홀더 전략("절대 stock photo를 사용하지 않는다")이 브랜드 품위를 보호한다.
- 모션 원칙에서 "물속에서의 움직임을 참조한다: 부력, 저항, 느린 관성"이 브랜드 세계관과 직결된다.

**수정 요청**:
- 컬러 섹션(Section 4)에서 "TBD"를 명확히 했으나, 방향성으로 제시한 "깊은 골드 또는 머금은 듯한 앰버 계열" 악센트는 현재 참조용 hex값(#C4956A 등)과 일치하는지 확인 필요. 컬러 확정 시 총괄팀과 반드시 사전 협의할 것.
- 압력 체험 효과 적용 범위를 S1~S3으로 정의했으나 UX팀은 S2~S5로 정의했다. 아래 충돌 해결 참조.

### 섹션 2 — 관찰 (Observation)

**확정 카피**:
```
바다는 기록한다.

2024. 11. 03.
수온 3.2도. 조류가 멈추었다.
병이 바닥에 닿은 지 47일째.
표면의 진동이 여기까지 오지 않는다.

2025. 02. 18.
압력이 코르크를 누른다.
안쪽에서 기포가 느려진다.
시간이 액체에 스며드는 소리가 있다면
이런 소리일 것이다.

2025. 06. 01.
해류가 병을 0.3도 기울였다.
효모가 반응한다.
바다가 숙성을 결정한다. 우리가 아니다.

2025. 10. 27.
인양 전날.
병에 손을 대면 차갑다.
바다의 시간이 손끝에 전해진다.
```

**CTA**: "기록을 따라가다 →"

**UX 플로우**: 에디토리얼 매거진 레이아웃. 비대칭 2컬럼(4:8). 좌측 날짜/라벨 sticky, 우측 본문 스크롤. 각 엔트리 IntersectionObserver 0.3 트리거, fade-in-up 0.8s.

**비주얼**: 한지 텍스처 배경(opacity 3~8%). 필름 그레인 처리된 해저 사진 2~3장. 이미지 border-radius: 0.

**기술**: Framer Motion whileInView + useTransform(parallax ±40px). 모바일: parallax 해제, 엔트리 4개→3개 축소.

### 섹션 3 — 데이터 아카이브 (Data Archive)

**확정 카피**:
```
단순히 바다에 넣은 것이 아니다.
설계된 것이다.

수심 60m. 수온 1.8-3.4도.
압력 7.1기압.
365일 무진동 환경.

지상의 셀러가 만들 수 없는 조건을
바다가 만든다.
```

**데이터 라벨**:
```
수온 — temperature (°C)
압력 — pressure (atm)
조류 속도 — current velocity (m/s)
```

**CTA**: "데이터를 탐색하다 →"

**UX 플로우**: 중앙 정렬 8컬럼. 기하학 파동 3레이어 SVG(수온/압력/해류 데이터 매핑). 스크롤 진입 시 stroke-dashoffset 드로잉. 데이터 수치 카운팅(0→target, 2.0s). 호버 시 해당 파동 라인 하이라이트.

**기술 결정**: 깊이 선택 수직 슬라이더(프론트엔드팀 제안) 제거. 호버/탭 하이라이트 방식으로 단순화. GSAP ScrollTrigger + CSS @keyframes 혼합. dynamic import(ssr: false).

**모바일**: 파동 3라인→2라인, 호버→탭, 미니차트 숨김, hardwareConcurrency < 4 시 정적 SVG.

### 섹션 5 — 컬렉션 (Archive)

**확정 컬렉션명 (5-depth)** — 카피팀 안 승인:

```
— 20m   surface murmur.
         수면 가까이. 빛이 닿는 마지막 깊이.

— 30m   quiet pressure.
         빛이 사라지기 시작한다. 압력이 대화를 시작한다.

— 40m   blind current.
         조류만이 움직인다. 눈을 감아도 같다.

— 50m   still archive.
         모든 것이 느려진다. 시간이 퇴적된다.

— 60m   the bed.
         바닥이다. 여기서 기다린 것만이 올라간다.
```

**승인 근거**: 영문 소문자 통일, 각 이름이 해당 깊이의 감각적 특징을 함축한다. surface murmur(수면의 속삭임)에서 the bed(바닥)까지의 하강 구조가 브랜드 세계관과 일치한다. 과시하지 않고, 설명하지 않고, 존재 자체로 말한다.

**인증서 카피**: 확정. 공적 문서의 톤을 유지하되 최소 정보만 노출.

**QR 코드 안내**: 확정. "잠들어 있던"이라는 표현이 병을 생명체처럼 다루는 점이 아카이브 디스커버리 축에 부합.

**테이스팅 리추얼**: 확정. 시적 안내 형식. 마침표로 끊어 호흡.

**UX 플로우**: 데스크톱 수평 스크롤 캐러셀(sticky pin, 5카드 × 화면너비). 깊이 인디케이터(하단 5도트). 깊이가 깊어질수록 배경 brightness 감소(1.0→0.80). 모바일: 수직 카드 스택. 인증서: 호버 시 확대, 클릭 시 라이트박스.

**CTA**: "컬렉션을 열다 →" (카피팀) 또는 "발견하기 →" (UX팀). **결정: "발견하기 →"로 통일.** 이유: "발견하기"가 아카이브 디스커버리 축에 더 직접적으로 부합하며, 히어로 섹션의 CTA와 톤을 맞출 수 있다. 각 카드에 반복 배치되므로 짧은 표현이 적합하다.

## 팀 간 충돌/불일치 사항

### 1. 컬렉션 깊이 수치 불일치 (심각)

| 출처 | 깊이 수치 |
|------|-----------|
| 카피팀 | 20m, 30m, 40m, 50m, 60m |
| 프론트엔드팀 | 15m, 30m, 50m, 80m, 120m |
| PLAN.md | "수심 60m" 기준 (구체적 라인업 미지정) |
| UX팀 | "수심 15m" 예시 사용 (첫번째 깊이) |

**결정**: 카피팀의 20m~60m 채택. 이유:
1. PLAN.md의 "수심 60m" 기준과 정합.
2. 20m 간격이 직관적이고 기억하기 쉬움.
3. 120m은 실제 해저 숙성 가능 깊이 대비 과장될 수 있음.
4. 카피팀의 컬렉션명(surface murmur ~ the bed)이 이 깊이 체계와 이미 연결되어 있음.

**필요 조치**: 프론트엔드팀 인터페이스(`depth_m: 15 | 30 | 50 | 80 | 120`)를 `20 | 30 | 40 | 50 | 60`으로 수정. UX팀 와이어프레임 내 수심 표기 수정.

### 2. Ocean Circle 폼 필드 불일치

| 출처 | 폼 필드 |
|------|---------|
| 카피팀 | 이름, 연락처, 바다에 남길 한 문장(선택) |
| 마케팅팀 | 이메일, 이름(성), 관심분야(선택: 태그) |
| 프론트엔드팀 | 이름, 이메일, 전화번호(선택), 개인정보동의 |
| UX팀 | 이메일, 이름 |

**결정**: 다음으로 통일:
```
이메일 (필수) — 유일한 커뮤니케이션 채널
이름 (필수) — 개인화된 초대장
바다에 남길 한 문장 (선택) — 의례적 장치, DB 수집이 아닌 경험
개인정보 수집 동의 (필수) — 법적 요구사항
```

**채택 이유**:
- 이메일이 핵심 채널이므로 필수.
- 전화번호는 프리런칭 단계에서 불필요. 추후 필요 시 추가.
- 마케팅팀의 "관심분야 태그"는 유용하나, 폼 필드 최소화 원칙에 따라 삭제. 세그먼트 분류는 런칭 후 행동 데이터로 대체.
- 카피팀의 "바다에 남길 한 문장"은 기능적으로 불필요하나, Ocean Circle을 단순 DB 수집이 아닌 의례로 만드는 장치로서 브랜드 가치가 높다. 선택 사항으로 채택.

### 3. 압력 체험 효과 적용 범위 불일치

| 출처 | 적용 범위 |
|------|-----------|
| UX팀 | S2~S5 (Observation ~ Archive) |
| 디자인팀 | S1~S3 (The Void ~ Data Archive) |
| PLAN.md | 담당: 프론트엔드 + UX (범위 미지정) |

**결정**: UX팀의 S2~S5 채택.

**이유**:
1. S1(Hero)은 고유한 3단 전환 체계가 있으므로 별도 압력 효과를 중첩하면 과부하.
2. S2~S5가 "해저 여정"의 핵심 구간이며, S6(Ocean Circle)부터는 수면 위로 올라오는 감정이므로 압력 해제가 자연스럽다.
3. UX팀이 "S6~S7은 폼 영역이므로 입력 정확도 보존"을 이유로 제외한 판단이 합리적이다.

**디자인팀 요청**: 적용 범위를 S2~S5로 수정하되, 비주얼 표현(텍스처 scale 변화, 파동 진폭 변화)은 디자인팀 방식 유지.

### 4. Section 2(Observation) 레이아웃 차이

| 출처 | 레이아웃 |
|------|---------|
| UX팀 | 비대칭 2컬럼 (col 2~5 이미지 + col 7~11 텍스트), 교차 배치 |
| 디자인팀 | 비대칭 2컬럼 (4col 라벨 + 8col 콘텐츠), 좌측 sticky 라벨 |

**결정**: 디자인팀의 "좌측 sticky 날짜/라벨 + 우측 본문" 구조를 기본으로 하되, UX팀의 풀블리드 이미지 배치를 병합한다.

**이유**: 좌측 날짜가 sticky로 고정되는 것이 "관찰 일기"의 시간 축을 시각적으로 강화한다. UX팀의 이미지-텍스트 교차 배치는 에디토리얼 감각이 좋지만, 일기 형식의 일관성을 위해 "날짜 고정 + 본문/이미지 수직 진행" 구조가 더 적합하다.

### 5. Section 5(Archive) 레이아웃 차이

| 출처 | 데스크톱 레이아웃 |
|------|---------|
| UX팀 | 수직→수평 스크롤 캐러셀 (sticky pin, 5카드 × 화면너비) |
| 디자인팀 | 수평 스크롤 또는 1행 5열 그리드 (각 카드 280px) |

**결정**: UX팀의 sticky 수평 스크롤 캐러셀 채택.

**이유**: 5개 깊이를 "탐험"하는 경험을 구현하려면 수직 스크롤이 수평 이동으로 전환되는 몰입감이 필요하다. 1행 5열 그리드는 정보를 한눈에 보여주지만, "발견"의 감각이 약하다. 다만 모바일에서는 UX팀 제안대로 수직 카드 스택으로 전환한다.

### 6. B2B 응답 시간

| 출처 | 응답 시간 |
|------|-----------|
| UX팀 | "영업일 기준 2일" |
| 마케팅팀 | "3영업일 이내" (일반), A등급 24h / B등급 48h / C등급 72h |

**결정**: UI에는 "영업일 기준 2일 내"로 표기. 내부 운영은 마케팅팀의 등급별 기준(A: 24h, B: 48h, C: 72h) 적용.

**이유**: 고객 대면 메시지에서 "2일"이 "3일"보다 신뢰감 있다. 내부적으로 등급 분류 후 A등급은 24시간 내 응답하므로 실질적으로 2일 약속을 지킬 수 있다.
