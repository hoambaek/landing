# Observation 섹션 — 크리에이티브 재구상

> **"웹사이트가 아니라 설치미술이다."**
>
> 기존 분석(`observation-compact-analysis.md`)은 400vh를 250vh로 줄이는 데 성공했다.
> 그러나 그것은 같은 언어를 더 작은 목소리로 말한 것일 뿐이다.
> 이 문서는 다른 언어를 제안한다.

---

## 전제

사용자는 히어로 영상을 통해 이미 물속에 들어왔다.
수면을 관통하는 POV 영상이 끝났다. 숨이 차다.
이제 **가라앉는 일만 남았다.**

보존해야 할 것:
```
0m      수면. 빛과 공기.
—10m    빛의 90%가 사라지는 지점.
—20m    3기압. 기포의 직경이 절반으로.
—30m    수온 8°C. 숙성이 시작되는 온도.
—40m    완전한 어둠. 완전한 고요. 그리고 26,280시간의 기다림.

        시간이 닿지 않는 곳에서,
        시간만이 할 수 있는 일.
```

파괴해야 할 것:
- 카드. 타임라인. 아코디언. 스크롤재킹. 점진적 페이드인.
- "스크롤하면 텍스트가 나타난다"는 2016년의 문법.
- 사용자가 한 번이라도 본 적 있는 모든 것.

## A. 압력 타이포그래피 — *Pressure Typography*

### 콘셉트

텍스트가 물이다. 글자 자체가 수압을 받는다.

0m에서 카피는 넉넉한 자간과 행간으로 호흡한다. 스크롤할수록 글자가 물리적으로 찌그러진다. letter-spacing이 음수로 진입한다. line-height가 1.0 아래로 내려간다. 글자와 글자가 겹치기 시작한다. —40m에서 텍스트는 거의 하나의 검은 덩어리가 된다. 읽을 수 있지만, 읽는 행위 자체에 저항이 생긴다.

클라이맥스에서 — 압축된 덩어리가 갑자기 폭발하듯 원래 자간으로 돌아온다. 압력이 해제되는 순간. 병뚜껑이 열리는 순간의 메타포.

### 영감

— **Jenny Holzer**, LED 설치작. 텍스트가 물리적 공간을 점유하는 방식.
— **Concrete Poetry** 운동. 시의 의미가 배치 자체에 있다는 믿음.
— **John Cage**, 4'33". 읽을 수 없는 것도 읽기의 일부다.

### 시각적 묘사

**0m** — 화면 중앙. `font-size: clamp(48px, 8vw, 120px)`. `letter-spacing: 0.15em`. `line-height: 2.4`. 글자 사이에 바다가 흐른다. 배경: `--warm-ivory`.

**—10m** — 같은 위치. `letter-spacing: 0.06em`. `line-height: 1.8`. 글자가 조금 가까워졌다. 배경이 5% 어두워졌다. 사용자는 아직 변화를 '인지'하지 못한다. 무의식이 먼저 반응한다.

**—20m** — `letter-spacing: 0em`. `line-height: 1.4`. 글자가 서로 닿기 시작한다. "3기압"의 '3'과 '기'가 거의 붙어 있다. "기포의 직경이 절반으로" — 이 문장 자체가 절반의 공간에 압축되어 있다. 배경: 중간 그레이.

**—30m** — `letter-spacing: -0.03em`. `line-height: 1.1`. 글자가 겹친다. "수온 8°C"의 숫자와 도 기호가 하나의 형태가 된다. 텍스트가 읽히기 직전의 경계. 배경: `--void-bg`의 80%.

**—40m** — `letter-spacing: -0.06em`. `line-height: 0.85`. 글자들이 하나의 블록으로 합쳐진다. "완전한 어둠"이라는 문장이 실제로 어둠처럼 보인다 — 글자의 형태가 사라지고 질량만 남는다. 배경: `--void-bg`.

**클라이맥스** — 2초의 정적. 그리고 `letter-spacing: 0.2em`으로 복귀. `line-height: 3.0`. "시간이 닿지 않는 곳에서," — 글자 하나하나 사이에 침묵이 있다. 압력이 풀린 순간의 해방감.

### 기술 구현

```
- 엔진: Framer Motion useScroll + useTransform
- 핵심 보간: scrollYProgress → letter-spacing, line-height, font-weight(가변폰트 사용 시)
- 배경: CSS custom property 보간 (--depth-progress: 0~1)
- 글자 겹침: CSS만으로 가능. JS 불필요.
- 클라이맥스 해제: Framer Motion spring 트랜지션 (stiffness: 80, damping: 12)
- 스크롤 높이: 200vh (핀). 5단계 progress 구간.
- 텍스트 렌더링: 각 글자를 <span>으로 분리하여 개별 transform 가능 (선택적)
- 접근성: prefers-reduced-motion 시 정적 레이아웃으로 폴백. 모든 텍스트 즉시 표시.
```

### 감정 곡선

```
0m    : 고요. 넓은 호흡. "아름답다."
—10m  : 미세한 불안. "뭔가 달라졌다."
—20m  : 압박. "글자가 눌린다." 신체적 반응 시작.
—30m  : 긴장. "읽기 어렵다." 앞으로 기울이게 된다.
—40m  : 질식감. "텍스트가 사라지고 있다."
해제  : 카타르시스. 숨을 내쉰다.
```

### 컴팩트성

200vh. 핀 섹션. 5단계 사이의 전환이 '공간'이 아니라 '밀도'로 표현되므로 빈 화면이 없다.

### 리스크

- 가독성. —30m 이후 텍스트가 실제로 읽기 어려워지는데, 이것이 의도인지 결함인지 사용자가 판단하지 못할 수 있다.
- 해결: 수심 카운터(`—30m`)는 항상 가독성 유지. 카운터가 "여기는 깊다"는 앵커 역할.
- 모바일에서 음수 letter-spacing이 한글 렌더링에 미치는 영향 테스트 필요.

## B. 호흡 동기화 — *Respiratory Interface*

### 콘셉트

화면이 숨을 쉰다. 사용자도 함께 쉰다.

섹션 진입 시 화면 전체가 미세하게 팽창하고 수축한다. CSS scale과 opacity의 맥동. 각 깊이는 하나의 호흡 사이클에 대응한다. 0m에서 호흡은 빠르고 얕다(3초 사이클). —40m에서 호흡은 느리고 깊다(8초 사이클). 사용자의 실제 호흡이 화면의 리듬에 동기화된다.

스크롤이 아니다. **시간**이다. 사용자는 스크롤하지 않는다. 화면이 호흡할 때마다 다음 깊이로 자동 전환된다. 스크롤은 "호흡을 건너뛰는" 행위가 된다.

### 영감

— **Bill Viola**, *The Crossing* (1996). 극도로 느린 시간이 신성을 획득하는 방식.
— **James Turrell**, *Breathing Light*. 빛이 호흡하는 공간.
— **Marina Abramovic**, *The Artist is Present*. 함께 존재하는 시간의 무게.

### 시각적 묘사

**진입** — 화면 전체가 `--warm-ivory`. 중앙에 아무것도 없다. 2초. 그리고 화면이 미세하게 밝아졌다 어두워진다. 숨을 들이쉬고 — 내쉬고. 첫 번째 호흡의 정점에서 "수면. 빛과 공기."가 드러난다.

**0m → —10m** — 호흡 3사이클(약 9초). 매 호흡마다 화면의 밝기 범위가 좁아진다. 밝은 정점이 조금씩 낮아지는 것. 세 번째 호흡의 저점에서 "빛의 90%가 사라지는 지점."

**—20m → —30m** — 호흡 사이클이 5초로 느려진다. 화면의 맥동이 더 깊어진다. scale이 개입 — 들숨에서 화면이 0.2% 팽창, 날숨에서 수축. 이것은 수압의 물리적 표현.

**—40m** — 호흡 사이클 8초. 거의 명상적. 화면이 완전히 어둡다. 호흡의 밝기 범위가 `#0A0908` ~ `#0D0B09` — 거의 구별 불가능한 미세한 차이. 그 안에서 텍스트가 나타나고 사라진다.

**클라이맥스** — 호흡이 멈춘다. 완전한 정지. 3초. 그리고 "시간이 닿지 않는 곳에서," 화면이 한 번 크게 숨을 들이쉰다.

### 기술 구현

```
- 엔진: GSAP 타임라인 + Framer Motion
- 호흡 사이클: GSAP 타임라인의 repeat(-1)로 무한 루프.
  gsap.to(container, { scale: 1.002, opacity: breathMax, duration: breathDuration / 2, yoyo: true, repeat: -1 })
- 깊이 전환: IntersectionObserver 또는 타이머 기반. 스크롤 보조.
- 호흡 속도 변화: breathDuration을 scrollYProgress에 매핑 (3s → 8s)
- 배경: 호흡 맥동 = opacity 변조. 기본 배경색은 progress로 보간.
- 텍스트 등장: 호흡 정점(scale max)에서 fade-in. 호흡 저점에서 fade-out. 자연스러운 리듬.
- 스크롤 높이: 250vh (핀). 스크롤이 가능하지만, 멈춰도 호흡은 계속된다.
- 성능: GPU 가속 (will-change: transform, opacity). rAF 기반.
- 접근성: prefers-reduced-motion 시 호흡 비활성, 정적 텍스트 시퀀스로 전환.
```

### 감정 곡선

```
진입  : 혼란. "왜 아무것도 없지?" 그리고 깨달음. "숨을 쉬고 있다."
0m    : 동기화. 자신의 호흡이 화면과 맞기 시작한다.
—20m  : 느려짐. 몸이 이완된다. 명상적 상태 진입.
—40m  : 초월. 화면과 내가 하나의 유기체.
해제  : 호흡 정지의 충격. 그리고 해방.
```

### 컴팩트성

250vh. 그러나 체감 시간은 45~60초. 시간이 공간을 대체한다.

### 리스크

- 사용자 이탈. 자동 전환 요소가 있어 "갇힌 느낌" 유발 가능.
- 해결: 스크롤로 언제든 건너뛸 수 있도록 허용. 호흡은 장식이지 장벽이 아니다.
- iOS Safari에서 scale 맥동이 layout shift를 유발할 수 있음. transform만 사용하여 방지.
- 호흡 동기화가 불편한 사용자(불안장애 등) 존재. `prefers-reduced-motion` 외 별도 토글 고려.

## C. 빛의 고갈 — *Light Subtraction*

### 콘셉트

빛을 더하는 것이 아니라 빼는 것.

화면은 순백으로 시작한다. 텍스트도, 이미지도, UI도 없다. 오직 빛. 스크롤하면 빛이 사라진다. 문자 그대로 — Canvas 위에 수만 개의 밝은 입자가 있고, 스크롤할수록 입자가 하나씩 꺼진다. 텍스트는 빛이 사라진 자리에 나타난다. 어둠이 글자의 형태이다. 빛이 빠진 구멍이 곧 메시지이다.

### 영감

— **James Turrell**, *Roden Crater*. 빛 자체가 매질이 되는 경험.
— **Anish Kapoor**, *Void*. 어둠이 물리적 깊이를 갖는 착시.
— **Ryoji Ikeda**, *data.tron*. 빛의 입자가 정보를 운반하는 방식.

### 시각적 묘사

**0m** — 전체 화면이 백색 입자로 가득하다. Canvas 2D 위에 30,000~50,000개의 미세한 점. 각 점은 `rgba(255, 255, 252, 0.7~1.0)`. 점들이 미세하게 떨린다 — 수면의 빛 반사. 화면 하단 중앙에 "수면. 빛과 공기." 텍스트가 입자 사이에 은밀하게 존재한다.

**—10m** — 입자의 90%가 소거된다. 실제로 90%. "빛의 90%가 사라지는 지점"이라는 카피가 시각적으로 증명되는 순간. 남은 10%의 입자가 화면 여기저기에 흩어져 있다. 텍스트는 어두운 배경 위에 오프화이트(`--void-text`)로 나타난다.

**—20m** — 입자 3%만 잔존. 화면 가장자리에서 가끔 깜박인다. "3기압. 기포의 직경이 절반으로." — 남은 입자의 크기가 절반으로 줄어든다. 데이터와 시각이 동기화.

**—30m** — 입자가 1% 미만. 더 이상 빛이라 부를 수 없다. 산발적인 형광 점멸. 심해 생물발광의 암시.

**—40m** — 입자 0개. 완전한 어둠. `--void-bg` 단색. "완전한 어둠. 완전한 고요." — 이 문장만이 유일한 빛이다. 글자 자체가 미세하게 발광한다. `text-shadow: 0 0 30px rgba(241, 239, 235, 0.08)`.

**클라이맥스** — 어둠 속에서 한 줄. "시간이 닿지 않는 곳에서," — `--amber` 색상의 미세한 글로우. "시간만이 할 수 있는 일." — 글자 주변에 앰버 골드 입자가 천천히 생성된다. 빛의 귀환. 그러나 처음의 차가운 백색이 아니라, 따뜻한 골드.

### 기술 구현

```
- 엔진: Canvas 2D (Three.js 불필요 — 단순 입자계)
- 입자 시스템:
  - 초기화: 30,000~50,000개 점. {x, y, radius, alpha, alive: true}
  - 소거: scrollYProgress에 따라 alive 비율 감소. Math.random() < killRate
  - 렌더링: rAF 루프. fillRect 또는 arc로 점 그리기.
  - 성능: 50K 점의 fillRect는 Canvas 2D로 60fps 가능 (GPU 부담 미미)
- 텍스트: Canvas 위 HTML 레이어 (position: absolute). Canvas는 배경.
- 스크롤 높이: 200vh (핀). 입자 소거 비율만 progress에 매핑.
- 클라이맥스 입자 재생성: 새로운 입자를 --amber 색상으로 spawn.
- 모바일: 입자 수를 15,000개로 축소. 성능 유지.
- 접근성: prefers-reduced-motion 시 Canvas 비활성, 배경색 그라데이션으로 대체.
```

### 감정 곡선

```
0m    : 경이. "화면이 빛으로 가득하다."
—10m  : 충격. "90%가 한순간에 사라졌다." 카피와 시각의 동기화가 주는 지적 쾌감.
—20m  : 공허. 빛이 거의 없다. 외롭다.
—30m  : 두려움. 어둠이 실체를 갖는다.
—40m  : 숭고. "완전한 어둠"이 진짜 어둠이다.
해제  : 희망. 골드빛이 돌아온다. 그러나 다른 종류의 빛이다.
```

### 컴팩트성

200vh. 입자 소거가 즉각적 시각 피드백을 제공하므로 "빈 화면" 문제 완전 해소.

### 리스크

- Canvas와 HTML 텍스트 레이어의 z-index 관리. 텍스트가 Canvas 위에 항상 표시되어야 한다.
- 50K 입자의 초기 렌더링이 모바일에서 첫 프레임 지연 유발 가능. 해결: 점진적 생성(첫 500ms 동안 0 → 50K).
- Canvas가 고정 크기라면 resize 대응 필요. ResizeObserver로 처리.

## D. 데이터 물성 — *Materialized Data*

### 콘셉트

숫자가 풍경이다.

0, 10, 20, 30, 40 — 이 다섯 개의 숫자가 화면의 전부다. 각 숫자가 `font-size: 80vw`로 화면을 가득 채운다. 숫자는 건축물이다. 숫자의 획 안에 카피가 마이크로 텍스트로 새겨져 있다. 숫자 '0'의 곡선을 따라 "수면. 빛과 공기."가 흐른다. 숫자 '40'의 직선 안에 "완전한 어둠"이 갇혀 있다.

스크롤하면 숫자가 압축된다. `scaleY`가 줄어든다. 납작해진다. 수압에 눌린 숫자. 동시에 배경이 어두워진다.

### 영감

— **Ryoji Ikeda**, *test pattern*. 데이터가 미학이 되는 순간.
— **Stefan Sagmeister**, *The Happy Show*. 숫자와 타이포가 환경이 되는 설치.
— **teamLab**, *Transcending Boundaries*. 글자가 물리적 공간을 채우는 방식.

### 시각적 묘사

**0m** — 화면 전체에 `0`. Cormorant Garamond, 80vw. `--earth` 색상. 숫자 '0'의 내부 공간(카운터)에 "수면. 빛과 공기."가 13px로 새겨져 있다. SVG clip-path로 숫자 형태 안에만 텍스트 표시.

**—10m** — 숫자가 `10`으로 전환. 이전 `0`이 `scaleY: 0.3`으로 납작해지며 사라지고, `10`이 등장한다. `1`의 세리프 획 위에 "빛의 90%가" — `0`의 원형 안에 "사라지는 지점."

**—20m** — `20`. 숫자가 `scaleY: 0.7`로 이미 눌려 있다. `2`의 곡선이 압축되어 각진 형태에 가까워진다. "3기압"이 `2`의 상단 곡선에, "기포의 직경이 절반으로."가 `0` 안에.

**—30m** — `30`. `scaleY: 0.5`. 숫자가 절반으로 찌그러졌다. `3`의 두 곡선이 거의 닫힌다. "수온 8°C"가 상단 곡선에, "숙성이 시작되는 온도."가 하단에. 배경: 80% 어둠.

**—40m** — `40`. `scaleY: 0.3`. 숫자가 거의 선(line)이 되었다. `4`의 대각선과 `0`의 원이 납작한 타원으로. "완전한 어둠. 완전한 고요." — 이 문장이 납작한 `40` 형태 안에 압축되어, 문자 그대로 읽기 어렵다. 배경: 완전한 어둠.

**클라이맥스** — `40`이 천천히 `scaleY: 1.0`으로 복원된다. 동시에 색상이 `--amber`로 전환. 숫자 안에 새겨진 텍스트가 "시간이 닿지 않는 곳에서, 시간만이 할 수 있는 일."로 교체된다. 숫자가 기념비가 된다.

### 기술 구현

```
- 엔진: Framer Motion layout + GSAP
- 거대 숫자: CSS font-size: 80vw. overflow: hidden으로 뷰포트 밖 잘림.
- 숫자 내 텍스트:
  - 방법 1: SVG <clipPath>에 텍스트 경로 → 안에 <foreignObject>로 HTML 텍스트
  - 방법 2: CSS background-clip: text를 역이용 — 숫자 형태의 마스크 안에 텍스트 레이어
  - 방법 3: Canvas 2D에서 globalCompositeOperation: 'source-in'으로 마스킹
  - 추천: 방법 2 (CSS 전용, 성능 최적)
- scaleY 압축: Framer Motion useTransform(scrollYProgress, [0, 1], [1.0, 0.3])
- 숫자 전환: AnimatePresence + layout 애니메이션
- 배경: progress 기반 컬러 보간 (기존 로직 재사용)
- 스크롤 높이: 250vh (핀). 각 숫자에 50vh 할당.
- 접근성: 숫자 내 마이크로 텍스트에 aria-label로 전문 텍스트 제공.
```

### 감정 곡선

```
0m    : 압도. 거대한 숫자. "이것은 데이터인가 예술인가."
—10m  : 탐구. 숫자 안에서 텍스트를 찾는 행위. 발견의 기쁨.
—20m  : 불안. 숫자가 찌그러진다. 정보가 물리적으로 변형된다.
—30m  : 절박. "읽어야 하는데 숫자가 눌리고 있다."
—40m  : 체념. 숫자가 선이 되었다. 정보는 형태에 갇혔다.
해제  : 복원. 골드빛 숫자가 기념비처럼 서 있다. 장엄.
```

### 컴팩트성

250vh. 숫자 자체가 시각적 스펙터클이므로 빈 공간이 존재하지 않는다.

### 리스크

- 80vw 폰트가 모바일에서 뷰포트를 넘치거나 레이아웃을 깨뜨릴 수 있음. clamp 필수.
- SVG clip-path 내 텍스트의 한글 렌더링 이슈. 방법 2(CSS)가 가장 안전.
- 숫자 전환 애니메이션이 부드럽지 않으면 전체 경험이 무너짐. 충분한 이징 시간 확보 필요.
- 접근성: 마이크로 텍스트(13px 이하)가 WCAG AA 기준에 미달할 수 있음. aria-label + sr-only 병행.

## E. 시간 왜곡 — *Temporal Distortion*

### 콘셉트

깊이가 시간을 먹는다.

0m에서 스크롤은 즉각적이다. 손가락이 움직이면 화면이 따라온다. —20m부터 스크롤에 저항이 생긴다. Lenis의 lerp 값이 0.1에서 0.03으로 떨어진다. 같은 거리를 스크롤해도 화면은 느리게 반응한다. —40m에서는 한 번의 스크롤 제스처가 해소되는 데 4초가 걸린다. 사용자는 물리적으로 깊이의 저항을 느낀다.

동시에 — 화면에 표시되는 시계가 있다. 디지털 카운터가 실시간으로 흐른다. 0m에서 1초 = 1초. —20m에서 1초 = 2초. —40m에서 1초 = 26,280시간(3년). 카운터가 미친 듯이 돌아간다. 시간이 왜곡되고 있다.

### 영감

— **Christian Marclay**, *The Clock* (2010). 실시간과 영화적 시간의 중첩.
— **Olafur Eliasson**, *Your Uncertain Shadow*. 지각의 왜곡이 의식을 변형하는 방식.
— **Leni Riefenstahl**에서 **Chris Marker**까지 — 영화에서 시간을 늘이고 줄이는 전통.

### 시각적 묘사

**0m** — 화면 우하단에 작은 시간 카운터. `00:00:00.00` 형식. 1초에 1초가 흐른다. 화면 중앙에 "수면. 빛과 공기." 배경: 밝음. 스크롤: 즉각 반응. Lenis lerp: 0.1.

**—10m** — 카운터가 1.5배속으로 흐른다. 사용자가 인지하기 어려운 속도 차이. "빛의 90%가 사라지는 지점." Lenis lerp: 0.08. 스크롤에 미세한 끈적함이 생긴다.

**—20m** — 카운터가 3배속. "3기압. 기포의 직경이 절반으로." Lenis lerp: 0.05. 스크롤이 물 속에서 움직이는 것처럼 느려진다. 화면의 모든 전환 — 텍스트 등장, 배경 변화 — 의 duration이 1.5배로 늘어난다.

**—30m** — 카운터가 10배속. 숫자가 빠르게 돌아간다. "수온 8°C. 숙성이 시작되는 온도." Lenis lerp: 0.03. 스크롤이 매우 무겁다. 모든 전환 duration: 2배.

**—40m** — 카운터가 폭주. `시간` 단위가 `일`로, `일`이 `월`로, `월`이 `년`으로 바뀐다. "26,280시간"이라는 숫자까지 카운팅한 뒤 멈춘다. Lenis lerp: 0.015. 한 번의 스크롤이 해소되는 데 3~4초. 사용자의 손은 이미 떠났는데 화면은 아직 움직이고 있다.

**클라이맥스** — 카운터가 `26,280:00:00`에서 정지. 2초간 정적. 그리고 카운터가 역재생 — 빠르게 0으로 돌아간다. 동시에 Lenis lerp가 0.1로 복귀. "시간이 닿지 않는 곳에서, 시간만이 할 수 있는 일." 스크롤이 자유로워진 순간의 해방감.

### 기술 구현

```
- 엔진: Lenis (lerp 동적 조절) + GSAP (카운터) + Framer Motion (텍스트)
- Lenis lerp 조절:
  const lenis = new Lenis({ lerp: 0.1 });
  // scrollYProgress에 따라 동적 변경:
  lenis.options.lerp = lerp(0.1, 0.015, depthProgress);
- 시간 카운터:
  - GSAP 카운터 애니메이션. gsap.to(counterRef, { innerText: targetHours, duration: ... })
  - 속도 배율: timeMultiplier = lerp(1, 26280*3600, depthProgress) — progress에 따라 지수적 증가
  - 표시: requestAnimationFrame에서 실시간 갱신
- transition duration 변조:
  CSS custom property: --time-scale: 1 → 2 (progress 기반)
  모든 transition-duration에 calc() 적용: transition-duration: calc(var(--base-duration) * var(--time-scale))
- 스크롤 높이: 200vh (핀). 스크롤 저항이 체감 높이를 늘리므로 실제 높이는 짧아도 된다.
- 접근성: prefers-reduced-motion 시 Lenis 저항 비활성, 카운터 정적 표시.
```

### 감정 곡선

```
0m    : 자연스러움. 아무 이상 없다.
—10m  : 의아함. "스크롤이 좀 다르다?" 미세한 위화감.
—20m  : 저항감. "무겁다." 물리적 감각이 개입.
—30m  : 투쟁. "왜 안 움직여." 좌절감 → 깨달음. "아, 깊어지고 있구나."
—40m  : 경외. 시간이 폭주하는 카운터. 3년이라는 숫자의 무게.
해제  : 자유. 스크롤이 가벼워진 순간의 물리적 쾌감.
```

### 컴팩트성

200vh. 그러나 Lenis 저항으로 인한 체감 스크롤 시간은 약 20~25초. 물리적 컴팩트 + 시간적 확장.

### 리스크

- Lenis lerp를 너무 낮추면(0.01 이하) 스크롤이 "고장난 것"으로 인식될 수 있음.
- 해결: 0.015를 하한선으로 설정. 스크롤 제스처 후 반드시 목표 위치에 도달하도록 보장.
- 시간 카운터의 빠른 갱신이 모바일에서 렌더링 부하 유발. 60fps 미만 시 카운터 갱신 주기를 100ms로 낮춤.
- 사용자가 "사이트가 느리다"고 판단할 리스크. 수심 카운터 + 시각적 피드백으로 "의도된 느림"을 전달해야 함.

## F. 소거 인터페이스 — *Erasure Interface*

### 콘셉트

모든 정보가 처음부터 보인다. 스크롤은 정보를 추가하지 않는다. 제거한다.

섹션 진입 시 5개의 카피가 모두 화면에 표시되어 있다. 동시에. 겹쳐서. 읽을 수 없다. 텍스트가 텍스트 위에 텍스트. 하나의 팔림프세스트(palimpsest). 스크롤하면 가장 바깥층(0m)부터 사라진다. 제거될 때마다 아래층이 드러난다. —40m에 도달하면 마지막 한 문장만 남는다. 순수한 것은 제거의 결과다.

### 영감

— **Robert Rauschenberg**, *Erased de Kooning Drawing* (1953). 지우는 행위가 창작이 된 순간.
— **William Kentridge**, 드로잉 애니메이션. 그리고 지우고 다시 그리는 과정 자체가 작품.
— **Cy Twombly**, 겹쳐 쓴 텍스트. 가독성을 포기한 대신 획득한 물성.

### 시각적 묘사

**진입** — 화면에 5개의 문장이 모두 표시된다. 같은 위치에. 같은 크기에. 서로 다른 opacity로 겹쳐 있다.
```
수면. 빛과 공기.
빛의 90%가 사라지는 지점.
3기압. 기포의 직경이 절반으로.
수온 8°C. 숙성이 시작되는 온도.
완전한 어둠. 완전한 고요. 그리고 26,280시간의 기다림.
```
이것은 하나의 얼룩이다. 텍스트의 물질성. 색상은 모두 `--earth` 기반이되, 다섯 문장이 미세하게 다른 opacity와 Y 오프셋으로 겹쳐 있어 두께감(depth)을 만든다. 배경: `--warm-ivory`.

**0m → 소거 시작** — 스크롤하면 맨 위층("수면. 빛과 공기.")이 사라진다. `opacity: 1 → 0` + `blur(4px) → blur(20px)`. 마치 물에 녹듯. 동시에 아래 문장들이 약간 선명해진다.

**—10m** — "빛의 90%가 사라지는 지점."이 홀로 가독 가능한 순간. 이전 문장은 이미 녹았고, 이후 문장들은 아직 흐릿하다. 2초. 그리고 이 문장도 녹기 시작한다.

**—20m, —30m** — 같은 패턴. 한 번에 하나의 문장만 선명. 나머지는 흐릿한 잔상이거나 이미 사라진 공백.

**—40m** — "완전한 어둠. 완전한 고요. 그리고 26,280시간의 기다림." 홀로 남는다. 위의 네 문장이 모두 사라진 자리에 이 한 문장. 화면의 90%가 비어 있다. 이 여백이 26,280시간이다.

**클라이맥스** — 마지막 문장도 사라진다. 완전한 공백. 3초. 그리고 새로운 텍스트가 다른 색상으로(`--amber`) 나타난다. "시간이 닿지 않는 곳에서, 시간만이 할 수 있는 일." 소거의 끝에 남은 진수.

### 기술 구현

```
- 엔진: Framer Motion
- 5개 텍스트 레이어:
  <motion.div style={{ opacity, filter: blur, y }}>텍스트</motion.div> × 5
  각 레이어의 opacity, blur, y를 scrollYProgress에 매핑
- 초기 상태:
  layer[0]: opacity 0.9, y: 0px
  layer[1]: opacity 0.75, y: 3px
  layer[2]: opacity 0.6, y: 6px
  layer[3]: opacity 0.45, y: 9px
  layer[4]: opacity 0.3, y: 12px
- 소거 순서: layer[0] → layer[1] → ... → layer[4]
  각 레이어: opacity → 0, blur → 20px (progress 구간별)
  선명화: 현재 활성 레이어의 opacity → 1.0, blur → 0
- 배경: progress 기반 컬러 보간 (밝음 → 어둠). 텍스트 색상도 동기화.
- 스크롤 높이: 200vh (핀).
- 성능: CSS filter: blur()는 GPU 가속. 5개 레이어 정도는 부하 미미.
- 접근성: 초기 겹침 상태에서 스크린리더는 순서대로 읽음 (aria-hidden 활용하여 현재 활성 텍스트만 노출).
```

### 감정 곡선

```
진입  : 혼란 + 호기심. "이게 뭐지? 읽을 수 없다."
0m    : 깨달음. "아, 스크롤하면 벗겨지는구나."
—10m  : 쾌감. 층이 벗겨지면서 드러나는 것의 만족감.
—20m  : 몰입. 다음 층을 보고 싶다.
—30m  : 긴장. "마지막이 뭘까."
—40m  : 고독. 하나의 문장만 남은 화면. 압도적 여백.
해제  : 골드빛 출현의 경이.
```

### 컴팩트성

200vh. 시각적 밀도가 처음부터 최대이므로 빈 화면이 원리적으로 불가능.

### 리스크

- CSS blur의 모바일 성능. 5개 레이어 동시 blur는 저사양 기기에서 지연 가능.
- 해결: will-change: filter 적용. 비활성 레이어(opacity 0)는 display: none으로 DOM에서 제거.
- 초기 겹침 상태가 "버그"로 보일 수 있음. 미세한 시각적 단서(예: 화면 하단의 스크롤 화살표)로 상호작용 유도.
- 5개 문장의 한글이 겹쳤을 때의 시각적 인상이 라틴 알파벳과 다를 수 있음. 한글 자소의 밀도가 높아 "얼룩"이 아니라 "검은 덩어리"가 될 위험.

## G. 음압 — *Negative Pressure*

### 콘셉트

화면이 안으로 빨려 들어간다.

물리적 수압은 바깥에서 안으로 누르는 힘이다. 이것을 CSS perspective와 transform으로 구현한다. 스크롤할수록 화면의 가장자리가 안쪽으로 휘어든다. 텍스트가 중앙으로 수렴한다. 뷰포트 자체가 찌그러지는 착시. —40m에서 모든 것이 화면 중앙의 한 점으로 수렴한다 — 블랙홀. 소실점.

### 영감

— **Anish Kapoor**, *Descent into Limbo* (1992). 바닥의 검은 원이 실제로 깊어 보이는 착시.
— **Olafur Eliasson**, *Your Blind Passenger*. 안개 속에서 공간감이 소멸하는 경험.
— **Stanley Kubrick**, *2001: A Space Odyssey* 스타게이트 시퀀스. 소실점으로의 가속.

### 시각적 묘사

**0m** — 화면이 평면이다. 일반적인 웹페이지. "수면. 빛과 공기." 중앙 정렬. 배경: `--warm-ivory`. 그러나 화면 사면에 미세한 비네팅(vignette) — 가장자리가 0.5% 어둡다. CSS radial-gradient.

**—10m** — `perspective: 2000px`. `transform: rotateX(0.5deg)`. 화면이 아주 미세하게 기울어진다. 상단이 살짝 뒤로. 원근감의 시작. 비네팅이 3%로 짙어진다. "빛의 90%가 사라지는 지점."

**—20m** — `perspective: 1200px`. 화면 좌우 가장자리에 `transform: scaleX(0.97)` — 가장자리가 안쪽으로. 중앙 콘텐츠는 정상. "3기압"의 텍스트 주변으로 화면이 오므라든다. 비네팅 10%.

**—30m** — `perspective: 800px`. 상하좌우가 안쪽으로 휘어든다. `border-radius: 5%`가 뷰포트에 적용된 듯한 착시 (실제로는 radial-gradient + 가장자리 마스크). 화면이 터널이 된다. 중앙의 텍스트만이 유일한 앵커.

**—40m** — `perspective: 400px`. 화면이 깊은 우물이다. 가장자리에서 중앙으로의 그라데이션이 극적. 중앙에 작은 원형 공간만 밝다(직경 30vw). 나머지는 어둠. "완전한 어둠. 완전한 고요." — 이 밝은 원 안에서만 읽을 수 있다. Kapoor의 *Void*처럼.

**클라이맥스** — 밝은 원이 점점 커진다. `30vw → 100vw`. 어둠이 밀려난다. 화면이 다시 평면으로. 그러나 배경은 `--void-bg`이고, 텍스트는 `--amber`. "시간이 닿지 않는 곳에서, 시간만이 할 수 있는 일." 우물 밖으로 나온 것이 아니다 — 우물 안에서 빛을 찾은 것이다.

### 기술 구현

```
- 엔진: CSS transform + Framer Motion + Canvas 2D (비네팅)
- 원근 효과:
  CSS perspective + translateZ. 컨테이너에 perspective, 텍스트에 translateZ(0) 고정.
  가장자리 요소에 translateZ(-50px ~ -200px)로 후퇴 효과.
- 비네팅:
  방법 1: radial-gradient(circle, transparent 30%, rgba(10,9,8, depth) 100%)
  방법 2: Canvas 2D로 동적 비네팅 (원형 밝은 영역의 크기를 progress로 조절)
  추천: 방법 1 (CSS 전용, GPU 가속)
- 터널 효과 (—30m~—40m):
  radial-gradient의 transparent 비율을 100% → 15%로 줄임
  = 밝은 영역이 점점 작아짐
- 텍스트: 항상 화면 중앙. perspective 영향을 받지 않도록 별도 레이어.
- 스크롤 높이: 200vh (핀).
- 성능: CSS transform + gradient만 사용. JS 연산 최소. 60fps 보장.
- 접근성: 비네팅은 장식. 텍스트 가독성은 항상 유지 (밝은 원 안에 텍스트 배치).
```

### 감정 곡선

```
0m    : 평온. 평범한 화면.
—10m  : 불안. "뭔가 기울어지고 있다." 무의식적 감지.
—20m  : 밀실공포. 화면이 좁아진다.
—30m  : 터널 비전. "빠져나갈 수 없다."
—40m  : 깊이의 숭고. 소실점의 아름다움.
해제  : 빛의 팽창. 물리적 해방감.
```

### 컴팩트성

200vh. CSS만으로 구현 가능하므로 추가 로딩 없음. 시각 효과가 연속적이라 빈 화면 없음.

### 리스크

- CSS perspective가 모바일에서 의도하지 않은 레이아웃 변형 유발 가능. 모바일에서는 perspective 대신 비네팅 강도만 조절하는 폴백 필요.
- "화면이 고장났다"는 인식 위험. 수심 카운터가 앵커 역할을 해야 함.
- 비네팅이 너무 강하면 가장자리 콘텐츠(헤더, 수심 카운터)가 보이지 않음. 이들은 비네팅 레이어 위에 배치.

## 추천

### 1순위: C. 빛의 고갈 — *Light Subtraction*

**이유:**

이 접근법만이 카피의 핵심 메시지를 *문자 그대로* 시각화한다.

"빛의 90%가 사라지는 지점" — 실제로 화면의 입자 90%가 사라진다.
"기포의 직경이 절반으로" — 남은 입자의 크기가 절반이 된다.
"완전한 어둠" — 입자가 0개가 된다.

텍스트가 묘사하는 것과 화면이 보여주는 것 사이에 간극이 없다.
이것은 "일러스트레이션"이 아니다. "증명"이다.

기술적으로도 Canvas 2D 입자계는 검증된 패턴이다. Three.js가 필요 없다. 번들 추가 없음. 모바일에서 입자 수를 줄이면 성능 보장 가능. `prefers-reduced-motion` 대응도 명확하다 — Canvas를 끄고 배경색 그라데이션으로 폴백.

클라이맥스에서 앰버 골드 입자의 재탄생은 브랜드 컬러(`--amber: #CCAD7B`)를 서사적으로 도입하는 완벽한 순간이다. 차가운 백색의 빛이 사라지고, 따뜻한 골드의 빛이 태어난다 — 이것이 해저 숙성의 메타포다.

### 2순위: A. 압력 타이포그래피 — *Pressure Typography*

**이유:**

가장 순수한 접근. 추가 기술 레이어(Canvas, WebGL) 없이 CSS와 Framer Motion만으로 구현 가능. 텍스트 자체가 매체. 여기서 "기술"은 보이지 않는다 — 오직 타이포그래피만 보인다. Editorial Minimalism의 가장 급진적인 확장.

C와 결합 가능: Canvas 입자 배경 위에서 텍스트가 압축되는 이중 레이어.

### 3순위: F. 소거 인터페이스 — *Erasure Interface*

**이유:**

"모든 것을 보여준 뒤 하나씩 빼간다"는 역발상이 강력하다. 일반적인 웹 경험(스크롤 → 콘텐츠 추가)의 정확한 반전. 기술적으로 가장 가벼움(HTML + CSS만). 단, 한글 텍스트의 겹침이 라틴 알파벳만큼 아름답지 않을 수 있어 프로토타입 검증 필수.

## 하이브리드 제안: C + A

> *빛이 사라지고, 글자가 찌그러진다.*

Canvas 입자 시스템(C)을 배경으로, 압력 타이포그래피(A)를 전경으로.
두 시스템이 동일한 `scrollYProgress`를 공유한다.

```
[레이어 구조]
z-3: Canvas 2D (입자 — 빛의 소거)
z-2: 배경색 보간 (밝음 → 어둠)
z-1: HTML 텍스트 (압축 타이포그래피)
z-0: 수심 카운터 (항상 가독)
```

**결과:**

0m에서 사용자는 빛의 바다 위에 떠 있는 넉넉한 글자를 본다.
—40m에서 사용자는 완전한 어둠 속에 으깨진 글자를 본다.
클라이맥스에서 앰버 입자가 피어나며 글자가 숨을 쉰다.

이것은 스크롤이 아니다.
하강이다.
