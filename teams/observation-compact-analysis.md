# Observation 섹션 컴팩트화 — 팀 합동 분석

> **목표**: 400vh 스크롤 높이를 축소하면서 "해저 하강" 감정 임팩트를 보존한다.
> **대상 섹션**: Observation (Pressure Descent + Part B 풀스크린 이미지)

---

## 1. 현재 상태 분석

### 수치 기반 현황

| 항목 | 현재 값 | 비고 |
|------|---------|------|
| 스크롤 높이 | 400vh | 약 12~16초 스크롤 시간 (일반 속도 기준) |
| 핀 영역 | 100vh (sticky) | 뷰포트 전체 고정 |
| 콘텐츠 요소 | Fact 5개 + Climax 1개 | 총 6단계 텍스트 시퀀스 |
| 배경 전환 | #ECEAE6 → #0A0908 | 밝음 → 완전 어둠 |
| 수심 카운터 | 0m → -40m | 연속 보간 |
| 라인 | 1px → 3px | 중앙 수직선 두께 변화 |
| 타이포 압축 | letter-spacing 0.04em→0.01em, line-height 2.2→1.7 | 압박감 표현 |
| Part B | 풀스크린 이미지 (f2.webp) | 스크롤 기반 reveal 없음, 단순 노출 |

### Fact 출현 구간 분석

현재 `factZoneStart=0.05`, `factZoneEnd=0.82`로 전체 progress의 77%를 Fact 표시에 사용한다.
각 Fact는 progress 기준 약 15.4% 구간을 차지하며, 이 중 40%가 등장(fade-in + slide-up), 50%가 퇴장(fade-out + slide-up) 구간이다.

**실제 스크롤 거리 환산** (400vh 기준):
- Fact 하나당: 약 61.6vh (= 400vh × 15.4%)
- 화면에 보이는 시간: 약 2.5~3초

### 이탈 리스크 진단

- **400vh = 4페이지 분량의 스크롤**을 단일 시각 요소(중앙 텍스트)로 유지해야 함
- 텍스트 간 전환 사이 "빈 화면" 구간 존재 — Fact 퇴장 후 다음 Fact 등장까지 공백
- 모바일에서 스크롤 속도가 빠르면 Fact를 놓칠 수 있음
- 럭셔리 사이트 벤치마크 기준, 단일 스크롤 핀 섹션의 권장 길이는 200~300vh (Krug: 약 250vh, Moët: 약 200vh)
- **콘텐츠 밀도**: 총 6개 텍스트 블록 / 400vh = 1.5 텍스트/100vh → 희소

## 2. 팀별 분석

### 01-frontend-team — 기술 접근법 비교

#### 접근법 A: 단순 높이 축소 (400vh → 200vh)

**구현**: `height: 400vh` → `height: 200vh`로 변경. 나머지 로직은 동일.

```css
.s-obs {
  height: 200vh; /* 400vh에서 축소 */
}
```

- **장점**: 변경 최소 (CSS 1줄). 기존 progress 계산, Fact 시퀀스, 배경 보간 모두 그대로 동작
- **단점**: 모든 효과의 속도가 2배. Fact 체류 시간이 약 1.2~1.5초로 줄어 읽기 전에 사라질 위험. 배경색 전환이 급격해져 "서서히 가라앉는" 느낌 약화
- **모바일**: 터치 스크롤 관성으로 Fact를 건너뛸 확률 높음
- **구현 난이도**: ★☆☆☆☆
- **권장 변형**: 200vh가 너무 빠르면 250vh~300vh 절충 가능. Fact 구간 비율 조정(`factZoneStart`/`factZoneEnd`)으로 체류 시간 보정

#### 접근법 B: 카드/스텝 기반 레이아웃

**구현**: 5개 Fact를 수직으로 쌓인 카드로 배치. 스크롤 핀 제거, 일반 document flow로 전환.

```
[Hero Section]
  ↓ (일반 스크롤)
[Card: 0m — 수면. 빛과 공기.]      ← 밝은 배경
[Card: -10m — 빛의 90%...]        ← 약간 어두운 배경
[Card: -20m — 3기압...]           ← 중간 어둠
[Card: -30m — 수온 8°C...]        ← 어두운 배경
[Card: -40m — 완전한 어둠...]      ← 거의 완전한 어둠
[Climax]                          ← 완전한 어둠
[Part B: 이미지]
```

- **장점**: 스크롤 핀 제거 → 자연스러운 스크롤. 각 카드에 IntersectionObserver로 개별 fade-in. 모바일 친화적. 각 카드 배경색을 단계별로 설정하여 그라데이션 효과 유지
- **단점**: "가라앉는" 연속 경험이 끊김. 수심 카운터 연속 보간이 어려움. 중앙 라인 효과 적용 난이도 상승. 현재 섹션의 독특한 시네마틱 느낌 상실
- **모바일**: 가장 안정적 — 일반 스크롤이므로 콘텐츠 누락 없음
- **구현 난이도**: ★★★☆☆ (전면 리라이트)
- **높이 추정**: 각 카드 80~100vh → 총 약 500vh이지만 스크롤 핀이 아니므로 체감은 짧음

#### 접근법 C: 타임라인 레이아웃

**구현**: 중앙 수직선을 유지하되, Fact를 좌우 교차 배치한 타임라인으로 전환. 일반 스크롤 + IntersectionObserver 기반 등장 애니메이션.

```
         │
    0m ──┤
         │── 수면. 빛과 공기.
         │
         ├── -10m
빛의 90% │
         │
         ├── -20m
         │── 3기압...
         │
```

- **장점**: 중앙 라인 메타포 보존. 한 화면에 여러 Fact 노출 가능 → 콘텐츠 밀도 증가. 럭셔리 브랜드 "아카이브" 느낌과 어울림 (Kinfolk 레퍼런스). 각 노드에 개별 배경 그라데이션 적용 가능
- **단점**: "한 번에 하나"라는 집중도 상실. 수심 카운터 연속 보간 어려움. 좌우 교차 레이아웃이 모바일에서 어색할 수 있음 (좁은 화면). 배경색 연속 전환이 "밴딩" 될 수 있음
- **모바일**: 좌우 교차 → 단일 열로 폴백 필요. 반응형 작업량 상당
- **구현 난이도**: ★★★★☆ (레이아웃 + 반응형 + 애니메이션 전면 변경)
- **높이 추정**: 약 250~350vh (Fact 간 여백에 따라)

#### 접근법 D: 어코디언/인터랙티브

**구현**: 5개 수심을 라벨로 나열, 클릭/호버 시 해당 Fact 펼침. 배경색은 선택된 수심에 따라 전환.

```
  0m   ←  ─10m  ─  ─20m  ─  ─30m  ─  ─40m
  ────────────────────────────────────────
  수면. 빛과 공기.
  [확장된 콘텐츠 영역]
```

- **장점**: 가장 컴팩트 (100vh 이하). 사용자 주도적 탐색
- **단점**: **럭셔리 브랜드 경험과 근본적으로 충돌**. "가라앉는" 수동적 경험이 "클릭해서 열기"라는 능동적 행위로 전환됨. 서사 흐름(0m→-40m) 순차성 보장 불가 — 사용자가 -40m을 먼저 볼 수 있음. 스크롤 기반 인터랙션이 아니므로 Lenis와의 시너지 상실
- **모바일**: 터치 어코디언은 흔한 패턴이지만 프리미엄 느낌 부족
- **구현 난이도**: ★★☆☆☆
- **높이 추정**: 약 80~100vh

#### 접근법 E: 싱글 뷰포트 애니메이션 (100vh 고정)

**구현**: 100vh 섹션 고정. 짧은 스크롤 핀(200vh)으로 5개 Fact를 빠르게 순환하거나, InView 트리거 시 자동 시퀀스(타이머 기반) 실행.

**변형 E-1 (짧은 핀)**:
```css
.s-obs { height: 200vh; } /* 핀 유지, 절반 축소 */
```
접근법 A와 유사하지만, Fact 전환 타이밍을 progress 비례 대신 easing 함수로 조정하여 체류 시간 확보.

**변형 E-2 (자동 시퀀스)**:
```
섹션 InView 진입 → 자동 재생:
0s: 0m fact (2초 체류)
2s: -10m fact (2초 체류)
...
10s: climax (3초 체류)
13s: 완료 → 다음 섹션으로 auto-scroll
```

- **장점**: 최소 높이. 자동 시퀀스 시 정확한 타이밍 제어 가능
- **단점**: E-2의 자동 재생은 사용자 스크롤 제어권 박탈 → 럭셔리 사이트에서 치명적 (Krug, Dom Pérignon 모두 사용자 주도 스크롤). E-1은 접근법 A의 변형
- **모바일**: 자동 재생 시 의도치 않은 스크롤 충돌 위험
- **구현 난이도**: E-1 ★★☆☆☆ / E-2 ★★★☆☆
- **높이 추정**: E-1: 200vh / E-2: 100vh

### 02-design-team — 시각 레이아웃 분석

#### "하강" 메타포의 시각적 핵심 요소

1. **연속적 어둠 전환** — 밝음에서 어둠으로의 점진적 변화가 물리적 깊이감을 생성
2. **중앙 수직선** — 하강 축. 두께 변화가 압력 증가를 암시
3. **수심 카운터** — 정량적 하강 지표. 숫자의 연속 변화가 "움직이고 있다"는 감각 제공
4. **한 번에 하나의 텍스트** — 고립된 정보 제시가 해저의 고독함과 일치
5. **타이포그래피 압축** — 물리적 압력의 시각적 은유

#### 접근법별 시각 전략

**접근법 A (높이 축소) — 시각 유지도: 95%**
모든 시각 요소 그대로. 전환 속도만 빨라짐. 배경 그라데이션의 중간 톤(회색-갈색 구간)이 짧아져 "점프" 느낌 발생 가능.
- 보정: 배경색 보간에 ease-in 커브 적용 → 초반 밝은 구간을 길게, 후반 어두운 구간을 빠르게

**접근법 B (카드) — 시각 유지도: 50%**
중앙 라인, 수심 카운터 연속 보간, 타이포 압축 모두 상실. 카드 간 배경색 단계 전환은 가능하나 "연속" 느낌이 "단계" 느낌으로 변환됨.
- 보정: 각 카드에 full-bleed 배경 + 카드 간 CSS gradient 전환 영역 삽입. 그러나 근본적 한계 존재

**접근법 C (타임라인) — 시각 유지도: 65%**
중앙 라인 보존. 그러나 동시에 여러 Fact가 보이므로 "한 번에 하나" 원칙 위반. 수심별 배경 전환은 섹션 단위로 가능.
- 보정: 각 타임라인 노드에 도달 시 해당 노드만 하이라이트, 나머지 dim 처리

**접근법 D (어코디언) — 시각 유지도: 30%**
연속 하강 경험 완전 상실. 정적 UI 요소로 전환됨. 럭셔리 비주얼과 괴리.

**접근법 E-1 (짧은 핀) — 시각 유지도: 90%**
A와 동일하되, easing 조정으로 중간 톤 체류 시간 확보 가능.

#### 컬러 전환 전략 (단축 시)

400vh → 200~250vh로 줄이면 배경 보간 곡선 조정이 필수:

```
현재 (선형):    ████░░░░░░░░████
               밝음  ────────  어둠

권장 (ease-in): ████████░░░░████
               밝음 길게 ── 어둠 빠르게
```

이유: 초반 밝은 구간에서 사용자가 섹션에 진입하고 적응하는 시간 확보. 어두워지는 구간은 가속하여 클라이맥스의 극적 효과 강화. 이는 실제 해저 광투과율 곡선과도 일치 (수면 근처에서 빛이 급격히 감소).

### 03-ux-team — 사용자 경험 분석

#### 스크롤 피로도 진단

| 지표 | 현재 (400vh) | 목표 (200~250vh) | 업계 평균 |
|------|-------------|-----------------|----------|
| 스크롤 시간 | 12~16초 | 6~8초 | 8~10초 |
| Fact 체류 시간 | 2.5~3초 | 1.5~2초 (조정 전) | 2~3초 권장 |
| 빈 화면 비율 | ~20% | ~10% (조정 후) | 5% 이하 권장 |
| 콘텐츠 밀도 | 1.5개/100vh | 2.5개/100vh | 2~3개/100vh |

#### 럭셔리 사이트 스크롤 패턴 벤치마크

- **Krug.com**: 핀 섹션 평균 250vh. 한 화면에 하나의 메시지. 텍스트 체류 최소 2초
- **Dom Pérignon**: 핀 섹션 200~300vh. 비디오/이미지 시퀀스와 텍스트 병행
- **Moët & Chandon**: 짧은 핀(150~200vh) + 일반 스크롤 섹션 교차
- **Kinfolk**: 핀 거의 미사용. 긴 여백 + 느린 IntersectionObserver fade-in

**시사점**: 럭셔리 샴페인 브랜드 사이트에서 400vh 단일 핀은 과도함. 250vh 전후가 "느리지만 지루하지 않은" 균형점.

#### 모바일 고려사항

- 터치 스크롤 관성 → 핀 섹션에서 "갇힌 느낌" 유발 가능. 높이 축소 시 완화
- 모바일에서 수심 카운터(`right: 80px`)가 콘텐츠와 겹칠 수 있음 → 이미 반응형 대응 확인 필요
- 짧은 핀(250vh 이하)이 모바일 터치 관성과 가장 잘 어울림
- iOS Safari의 주소창 축소/확장이 100vh 계산에 영향 → `dvh` 단위 고려

#### 인터랙션 프리미엄 패턴

럭셔리 사이트에서 "고급스러움"을 전달하는 인터랙션 특성:
1. **사용자 주도**: 스크롤 속도를 사용자가 결정. 자동 재생 지양
2. **예측 가능**: 다음에 무엇이 나올지 암시 (수심 카운터가 이 역할 수행)
3. **여운**: 콘텐츠가 충분히 머물다 자연스럽게 퇴장
4. **물리성**: 관성, 감속, 탄성 등 물리 법칙 모방 (Lenis가 담당)

→ 접근법 A/E-1이 이 네 가지 기준 모두 충족. B/C는 2~3개만 충족. D는 1개(사용자 주도)만 충족.

### 04-copywriting-team — 카피 구조 분석

#### 현재 5개 Fact 역할 분석

| 순서 | 수심 | 카피 | 서사 역할 |
|------|------|------|----------|
| 1 | 0m | 수면. 빛과 공기. | **출발점** — 익숙한 세계 설정 |
| 2 | -10m | 빛의 90%가 사라지는 지점. | **첫 번째 상실** — 빛 박탈 |
| 3 | -20m | 3기압. 기포의 직경이 절반으로. | **물리적 압력** — 과학적 구체성 |
| 4 | -30m | 수온 8°C. 숙성이 시작되는 온도. | **전환점** — 파괴에서 창조로 |
| 5 | -40m | 완전한 어둠. 완전한 고요. **그리고 26,280시간의 기다림.** | **도달** — 절대적 조건 + 시간 도입 |

이 5단계는 전형적인 **하강 서사(katabasis)**를 따른다: 일상 → 상실 → 압박 → 전환 → 도달.

#### 축소 가능성 평가

**결론: 5개 모두 유지를 권장한다.**

근거:
- 5개는 이미 최소 구조. 4개로 줄이면 서사 흐름에 구멍 발생
- 특히 Fact 3(3기압)과 Fact 4(수온 8°C)는 각각 "파괴"와 "창조"를 담당하므로 병합 시 의미 충돌
- Fact 1(0m)과 Fact 2(-10m)를 합치면 "출발점"과 "첫 번째 상실"이 동시에 제시되어 대조 효과 약화
- 컴팩트화는 카피 수정이 아니라 **전달 속도/레이아웃** 으로 해결해야 함

#### 클라이맥스 위치 재고

현재: 마지막 Fact(-40m) 이후 별도 구간(progress 85%~100%)에 표시.

**대안 검토**:
- 클라이맥스를 -40m Fact 직후에 빠르게 이어붙이면 연결감 강화. 단, 여백(쉼)이 줄어 여운 감소
- 현재 구조 유지를 권장. 마지막 Fact와 클라이맥스 사이의 "어둠 속 침묵"이 이 섹션의 정서적 정점

#### Fact 간 빈 구간 해소 제안

Fact 퇴장과 다음 Fact 등장 사이의 "빈 화면"이 현재 체감상 길다. 높이 축소 시 자연히 줄어들지만, 추가로:
- Fact 퇴장(fade-out)과 다음 Fact 등장(fade-in)을 약간 겹치게(overlap) 설정
- 현재 코드에서 `fEnd`와 다음 Fact의 `fStart`에 5~10% overlap 추가

## 4. 추천안

### 접근법 A 확장 — "조율된 높이 축소"

**핵심**: 400vh → 250vh로 축소 + Fact 타이밍 재조정 + 배경 보간 커브 변경

접근법 A(단순 축소)와 E-1(easing 조정)의 장점을 결합한다.

#### 추천 근거

1. **최소 변경 원칙**: 기존 코드의 구조, 로직, 클래스 모두 유지. CSS 높이 + JS 상수 4~5개만 수정
2. **검증된 구조 보존**: 현재 동작하는 sticky pin + progress 기반 시퀀스를 그대로 활용
3. **감정 임팩트 유지**: 모든 시각 효과(배경 전환, 라인, 카운터, 타이포 압축) 보존
4. **럭셔리 벤치마크 부합**: 250vh는 Krug(250vh), Moët(200vh)와 동일 범위
5. **리스크 최소**: 롤백 시 CSS 1줄 + JS 상수 복원으로 원상복귀

#### 구체적 변경 사항

**CSS**:
```css
.s-obs {
  height: 250vh; /* 400vh → 250vh */
}
```

**JS (ObservationSection.tsx)**:
```typescript
// Fact 구간 조정: 빈 화면 최소화
const factZoneStart = 0.03;  // 0.05 → 0.03 (빠른 시작)
const factZoneEnd = 0.80;    // 0.82 → 0.80 (유지)
const climaxStart = 0.83;    // 0.85 → 0.83 (간격 축소)

// Fact 간 overlap 추가: 퇴장-등장 겹침
const fPeak = fStart + ((factZoneEnd - factZoneStart) * 0.45) / factCount; // 0.4 → 0.45
const fEnd = fStart + ((factZoneEnd - factZoneStart) * 1.0) / factCount;   // 0.9 → 1.0
```

**배경 보간 커브**: 선형 → ease-in 변경
```typescript
// 기존: const progress = Math.max(0, Math.min(relScroll / (obsH - window.innerHeight), 1));
// 배경색 전용 progress에 ease-in 적용:
const bgProgress = progress * progress; // quadratic ease-in
const bg = lerpColor(LIGHT_BG, DARK_BG, bgProgress);
```

이 변경으로:
- 밝은 구간이 전체 스크롤의 약 60%를 차지 (현재 50%)
- 어두워지는 전환이 후반부에 가속 → 극적 효과 강화
- 실제 해저 광투과율 곡선(수면 근처 급감)과도 일치

## 5. 구현 플랜

### Phase 1: 높이 축소 + 확인 (변경 2건)

1. `globals.css` — `.s-obs { height: 250vh; }`
2. 브라우저에서 스크롤 체감 확인. Fact 읽기 시간이 2초 이상인지 검증

**완료 기준**: 모든 Fact가 데스크톱/모바일에서 읽기 가능한 속도로 표시됨

### Phase 2: Fact 타이밍 조정 (변경 3건)

1. `factZoneStart`, `factZoneEnd`, `climaxStart` 상수 조정
2. `fPeak`, `fEnd` 비율 조정 (Fact 간 overlap)
3. 브라우저에서 Fact 전환 검증. 빈 화면 구간이 체감상 0.5초 이하인지 확인

**완료 기준**: Fact 간 전환이 매끄럽고 빈 화면 없음. 클라이맥스까지 자연스러운 흐름

### Phase 3: 배경 보간 커브 (변경 1건)

1. `bgProgress = progress * progress` 적용
2. 텍스트 색상 전환도 동일 커브 적용 여부 판단 (시각 테스트 후 결정)

**완료 기준**: 밝음→어둠 전환이 자연스럽고, 중간 톤에서 "점프" 없음

### Phase 4: 모바일 검증

1. iOS Safari, Chrome Android에서 스크롤 테스트
2. 터치 관성 스크롤 시 Fact 누락 여부 확인
3. 필요 시 모바일 전용 높이 조정 (`@media` 분기)

**완료 기준**: 모바일에서 모든 Fact 읽기 가능. 갇힌 느낌 없음
