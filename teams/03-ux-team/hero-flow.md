## 개요

히어로 섹션은 "두 바다의 순환"이라는 핵심 서사를 설명 없이 전달하는 구간이다.
샹파뉴 토양(원래 바다) → 수면 관통 → 심해(현재 바다)의 여정을 사용자가 스스로 경험하게 설계한다.
강제하지 않는다. 압박하지 않는다. 사용자가 시간의 흐름 속으로 자연스럽게 침잠하도록 유도한다.

## 2. 각 단계별 사용자 경험 상태 및 인터랙션 상세

### Phase 1 — 지상 (Surface)

| 항목 | 내용 |
|------|------|
| **사용자 심리 상태** | 입장. 낯선 세계에 첫 발을 디딘다. 아직 지상이다. |
| **시각 신호** | 화석 산호의 촉각적 텍스처가 시간을 환기시킨다. |
| **주요 인터랙션** | 없음. 사용자는 관찰한다. 인터랙션을 강요하지 않는다. |
| **스크롤 인디케이터** | 가느다란 수직선 + 느린 펄스 애니메이션 (2s cycle). "내려갈 수 있다"는 신호만 제공. |
| **텍스트 체류 시간** | 페이지 진입 후 0.8s 딜레이 → 로고 페이드인 (1.0s) → 타이틀 페이드인 (0.6s) |
| **배경 이미지 처리** | 정적 이미지. 미세한 parallax 허용 (스크롤 depth 0.2). 움직임 최소화. |
| **앰비언트 사운드** | 기본값 음소거. 사운드 옵션 아이콘 우하단 배치 (아이콘만, 레이블 없음). |

**인터랙션 명세**
```
진입 시:
  0.0s  페이지 로드 완료
  0.8s  로고 opacity: 0 → 1 (duration: 1.0s, ease: ease-out)
  1.6s  타이틀 line 1 opacity: 0 → 1 (duration: 0.8s)
  2.2s  타이틀 line 2 opacity: 0 → 1 (duration: 0.8s)
  2.8s  스크롤 인디케이터 opacity: 0 → 0.6 (duration: 0.6s)

호버/포커스:
  없음. 텍스트는 인터랙티브 요소가 아니다.
```

### Transition — 수면 관통

| 항목 | 내용 |
|------|------|
| **사용자 심리 상태** | 경계를 넘는다. 익숙한 지상이 사라지고, 알 수 없는 깊이가 열린다. |
| **핵심 은유** | 물리적 침수 경험. POV(1인칭 시점)로 수면을 통과하는 것. |
| **트리거** | 스크롤 (wheel, touch, keyboard). 클릭/탭 기반 인터랙션 없음. |
| **스크롤 매핑** | `ScrollTrigger` 방식 — 스크롤 위치가 직접 애니메이션 진행도를 제어. |
| **역방향 지원** | 위로 스크롤 시 전환 역재생. 사용자가 경계를 자유롭게 넘나들 수 있다. |
| **물리적 피드백** | 스크롤 속도에 관계없이 전환은 일정한 속도 커브를 유지 (최대 가속 캡). |

**전환 시퀀스 세부 타임라인**

스크롤 진행도(progress)를 0.0 ~ 1.0으로 표기:

```
progress 0.00  Phase 1 완전 표시
progress 0.05  텍스트 그룹 fade-out 시작 (duration: progress 0.1 구간)
progress 0.15  텍스트 그룹 완전 소멸
progress 0.20  카메라 하강 시작 — 배경 이미지 translateY 음수 방향 이동
progress 0.25  수면 왜곡 레이어 fade-in 시작
               수면 경계선 등장 (하단 1/3 지점)
progress 0.35  색온도 전환 시작 — CSS filter hue-rotate/sepia 조합
               앰버(#EFDFBB 계열) → 틸(#3D5A56 계열)
progress 0.40  수면 경계선이 화면 중앙으로 이동
               지상 이미지 상단 클립 시작
               심해 영상 하단 클립 해제 시작
progress 0.55  수면 경계선이 화면 상단 밖으로 빠져나감
progress 0.60  심해 영상 autoplay 시작 (볼륨 0, 음소거)
               지상 이미지 opacity: 0 완료
progress 0.70  Phase 2 카피 텍스트 fade-in 준비 대기
progress 0.80  Phase 2 첫 번째 카피 fade-in 시작
progress 1.00  Phase 2 완전 표시
```

### Phase 2 — 심해 (Abyss)

| 항목 | 내용 |
|------|------|
| **사용자 심리 상태** | 침잠. 고요. 시간이 다르게 흐르는 공간에 존재한다. |
| **카피 연출** | 자동 진행 (스크롤 트리거 아님). 스크롤 고정(sticky) 상태에서 타이머 기반. |
| **카피 리듬** | 카피 1 등장(1.0s) → 체류(3.5s) → 퇴장(0.8s) → 공백(0.6s) → 카피 2 반복 |
| **영상 처리** | `preload="metadata"`, 인터섹션 옵저버로 뷰포트 진입 시 play, 이탈 시 pause |
| **CTA 등장** | 카피 3 완전 표시 후 1.5s 딜레이 → 텍스트 링크 fade-in |
| **CTA 언어** | "발견하기 →" (한국어), "Discover →" (영어). 버튼 형태 금지, 텍스트 링크 전용. |

**카피 타임라인 (Phase 2 sticky 진입 후)**
```
0.0s    Phase 2 sticky 시작
0.8s    카피 1 첫 줄 opacity: 0 → 1 (duration: 1.0s)
1.3s    카피 1 둘째 줄 opacity: 0 → 1 (duration: 1.0s)
4.8s    카피 1 fade-out (duration: 0.8s)
5.6s    (공백)
6.2s    카피 2 첫 줄 fade-in
6.8s    카피 2 둘째 줄 fade-in
10.8s   카피 2 fade-out
11.6s   (공백)
12.2s   카피 3 fade-in (단일 문장, 더 큰 타입)
13.2s   카피 3 체류 (고정)
14.7s   CTA "발견하기 →" fade-in (duration: 0.6s)
```

## 3. 전환 시퀀스 단계별 설명

### 3-1. 수면 관통 시각 효과 구현 전략

```
구현 방식: CSS clip-path + ScrollTrigger + SVG turbulence filter

단계 1 — 클립 분리
  지상 이미지: clip-path: inset(0 0 {50 + progress * 50}% 0)
  심해 영상:   clip-path: inset({100 - progress * 100}% 0 0 0)
  두 레이어가 스크롤 진행도에 따라 수직 분리점을 공유.

단계 2 — 수면 경계선
  SVG <feTurbulence> + <feDisplacementMap>로 물결 왜곡 생성.
  경계선 위치: transform: translateY({progress * -100}vh)
  turbulence baseFrequency: 스크롤 속도에 비례하여 0.01 ~ 0.04 동적 변경.

단계 3 — 색 전환
  배경 전체 CSS filter:
    hue-rotate(0deg → -30deg) + saturate(1.0 → 0.7) + brightness(1.0 → 0.75)
  심해 레이어 자체 색조: #3D5A56 계열 오버레이 opacity 0 → 0.35

단계 4 — 빛 굴절 표현
  수면 경계선 상하에 방사형 그라디언트 플레어:
    상단(지상): rgba(239, 223, 187, 0.3) 방사형
    하단(심해): rgba(61, 90, 86, 0.2) 방사형
  경계 통과 시 순간 brightness 1.15 피크 (렌즈 플레어 암시).
```

### 3-2. 스크롤 동작 세부 명세

```
전환 구간 스크롤 설정:
  pin: true (sticky 핀)
  scrub: 1.2 (스크롤과 애니메이션 간 0.2s 부드러운 지연)
  start: "top top"
  end: "+=200%" (전환 구간에 200vh 스크롤 여유)

스크롤 가속 캡:
  최대 scrub 진행 속도: 초당 progress 0.4
  급격한 스크롤 시 ease-out으로 감속하여 전환 완성도 유지.

키보드 지원:
  Space, Page Down: progress += 0.15 (애니메이션)
  Space, Page Up: progress -= 0.15 (역방향)
```

## 4. 모바일 대응 전략

### 4-1. 터치 인터랙션

```
기본 원칙: 스와이프 스크롤로 동일한 POV 잠수 경험을 제공한다.
           데스크톱과 경험의 본질이 같아야 한다. 축소판이 되어선 안 된다.

touch 이벤트 처리:
  - passive: true 리스너로 스크롤 성능 확보
  - touchmove velocity를 scrub에 전달 (관성 보존)
  - 핀치줌 제한 없음 (사용자 에이전시 존중)
```

### 4-2. 레이아웃 조정

| 요소 | 데스크톱 | 모바일 (< 768px) |
|------|----------|-----------------|
| 히어로 타이틀 | 48px / 중앙 정렬 | 32px / 중앙 정렬 |
| 카피 텍스트 | 24px / 중앙 정렬 | 18px / 좌측 패딩 24px |
| 스크롤 인디케이터 | 하단 중앙 | 하단 중앙 (위치 동일) |
| 배경 이미지 초점 | 화석 산호 전체 | 산호 클로즈업 중심 (object-position 조정) |
| 전환 구간 높이 | 200vh | 250vh (터치 스크롤 관성 보정) |
| 영상 해상도 | 1080p | 720p (대역폭 절감) |
| 카피 자동 타이밍 | 기본값 | 동일 (변경 없음) |

### 4-3. 성능 최적화 (모바일)

```
이미지:
  - Phase 1 배경: WebP, srcset 3단계 (1x/2x/3x), loading="eager"
  - 화면 진입 전 영상은 poster 이미지만 표시

영상:
  - <source> 두 개: mp4(H.264) + webm(VP9)
  - 모바일: 자동 720p 소스 선택 (Connection API 또는 화면폭 기준)
  - 데이터 절약 모드(Save-Data 헤더) 감지 시: 영상 대신 정적 프레임

전환 효과:
  - SVG turbulence 연산 비용 감지 → 저사양 기기 fallback:
    turbulence 대신 단순 linear-gradient 경계선으로 대체
  - 판단 기준: navigator.hardwareConcurrency < 4
```

## 5. 접근성 고려사항

### 5-1. prefers-reduced-motion 대응

```css
/* 감소 모션 설정 시 전략 */
@media (prefers-reduced-motion: reduce) {

  /* 전환 애니메이션 전면 비활성화 */
  .hero-transition-layer {
    animation: none;
    transition: none;
  }

  /* POV 잠수 효과 대신: 정적 크로스페이드 */
  /* Phase 1 → Phase 2: 스크롤 위치 기준 즉각 전환 (no scrub) */
  /* 경계선: 단순 수평 divider */

  /* 배경 영상 → 정적 포스터 이미지로 대체 */
  .phase-2-video {
    display: none;
  }
  .phase-2-poster {
    display: block; /* 영상의 대표 프레임 이미지 */
  }

  /* 카피 자동 진행 → 즉각 표시 (타이머 없음) */
  .abyss-copy { opacity: 1; }
}
```

**감소 모션 모드 사용자 경험**
- Phase 1은 정상 표시
- 스크롤 시 즉각(0.1s 크로스페이드) Phase 2로 전환
- 카피 3개 동시 표시 (순차 진행 없음)
- CTA 즉각 표시

### 5-2. 스크린 리더 지원

```html
<!-- Phase 1 -->
<section aria-label="Muse de Marée 히어로 — 지상" role="banner">
  <h1>Champagne mûri dans les abysses coréens. 한국의 심해에서 숙성한 샴페인.</h1>
  <!-- 배경 이미지: 장식용, aria-hidden="true" -->
</section>

<!-- Transition — 시각적 전환 구간, 스크린 리더 건너뜀 -->
<div aria-hidden="true" role="presentation">
  <!-- 수면 관통 애니메이션 레이어 -->
</div>

<!-- Phase 2 -->
<section aria-label="Muse de Marée — 브랜드 스토리" role="region">
  <!-- 영상: 장식용 -->
  <video aria-hidden="true" ...></video>

  <!-- 카피: 순차 제공 -->
  <p lang="ko">빛이 닿지 않는 곳에서 시간은 다르게 흐른다.</p>
  <p lang="ko">1만 년 전 산호의 숨결로 빚은 토양. 같은 바다로 돌아온 병.</p>
  <p lang="ko">두 바다의 순환.</p>

  <a href="/discover" aria-label="브랜드 스토리 전체 발견하기">발견하기 →</a>
</section>
```

### 5-3. 추가 접근성 명세

| 항목 | 기준 | 구현 방식 |
|------|------|----------|
| **색 대비** | WCAG AA (4.5:1 이상) | 텍스트/배경 대비 검증 필수. 어두운 오버레이로 보정. |
| **포커스 표시** | 키보드 포커스 링 필수 | CTA 링크: outline 2px #A7C7C0, offset 4px |
| **키보드 탐색** | Tab 순서 논리적 | Phase 1 → Phase 2 CTA → 이후 섹션 순 |
| **자동 재생 영상** | 음소거 필수 | `muted` 속성 하드코딩. 사용자 제어 버튼 제공. |
| **움직이는 콘텐츠** | 3회 이상 반복 금지 예외 | 영상 루프는 배경 장식으로 분류. `aria-hidden="true"` |
| **텍스트 크기** | 200% 확대 지원 | 뷰포트 단위 + rem 혼용. overflow 깨짐 QA 필수. |
| **영상 자막** | 대사 없는 앰비언트 영상 | 자막 불필요. 영상 목적 설명을 `aria-label`로 대체. |

## 6. 성공 기준

### 6-1. 기술 성능 KPI

| 지표 | 목표값 | 측정 방법 |
|------|--------|----------|
| **LCP (Largest Contentful Paint)** | ≤ 2.0s | Lighthouse, WebVitals |
| **CLS (Cumulative Layout Shift)** | ≤ 0.05 | 전환 구간 레이아웃 시프트 없어야 함 |
| **FID / INP** | ≤ 100ms | 스크롤 반응성 |
| **스크롤 프레임레이트** | 60fps 이상 유지 | Chrome DevTools Performance |
| **영상 로드 지연** | 수면 관통 도달 전 완료 | Network 탭 waterfail |
| **모바일 번들 크기** | JS ≤ 150KB (gzipped) | webpack-bundle-analyzer |

### 6-2. 브랜드 경험 KPI

브랜드 기준서(`brand-standards.md`) 심사 항목 전체 통과를 목표로 한다.

| 경험 기준 | 성공 상태 |
|----------|----------|
| **3초 안에 몰입** | Phase 1 진입 후 3초 내 브랜드 감도를 인지한다 |
| **설명 없는 서사 전달** | "두 바다의 순환" 개념을 텍스트 설명 없이 시각으로만 전달 |
| **자발적 발견 유도** | CTA를 누르기 전에 스크롤을 내리고 싶다는 욕구가 발생한다 |
| **압박 없는 경험** | 팝업, 강제 자동 전환, 알림 없음 |
| **전환 역방향 완성도** | 위로 스크롤 시 역전환이 순방향과 동일한 품질로 재생된다 |

### 6-3. 사용자 행동 지표 (정성)

프로토타입 사용자 테스트에서 확인할 항목:

```
□ 화석 산호 이미지를 보고 "바다"를 연상하는가?
□ 스크롤 인디케이터 없이도 자연스럽게 내려가는가?
□ 수면 관통 순간에 감각적 변화를 체감하는가? ("느낌이 달라졌다")
□ 심해 카피가 영상을 방해하지 않고 공존하는가?
□ "두 바다의 순환" 카피가 자연스럽게 수용되는가? (설명이 없어도)
□ CTA 클릭 전에 브랜드에 대한 정서적 연결이 형성되는가?
```

### 6-4. 접근성 성공 기준

```
□ WCAG 2.1 AA 준수 (자동화 검사 도구: axe-core)
□ prefers-reduced-motion: reduce 환경에서 완전한 기능 제공
□ 키보드만으로 CTA까지 도달 가능
□ 스크린 리더(VoiceOver, NVDA)에서 브랜드 카피 정상 낭독
□ 영상 자동재생 차단 환경에서 포스터 이미지 정상 표시
```
