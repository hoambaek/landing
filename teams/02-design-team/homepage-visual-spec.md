## 1. 비주얼 시스템 개요

### 디자인 철학

"더하지 말고 덜어낼 것. 여백이 메시지다."

이 페이지는 갤러리다. 방문자는 관객이다. 스크롤은 탐험이다. 지상에서 시작하여 심해로 내려가고, 다시 수면으로 올라오는 하나의 여정으로 설계한다. 각 섹션은 독립된 전시 공간이며, 섹션 전환은 물속에서 깊이가 바뀌는 감각을 전달한다.

에디토리얼 미니멀리즘이란: 잡지의 한 페이지처럼, 하나의 화면에 하나의 메시지만 존재한다. 텍스트와 이미지 사이의 넓은 여백이 호흡이 되고, 그 호흡 자체가 럭셔리를 구성한다.

### 무드 프로그레션 (섹션별 감정 아크)

```
Section 1 — The Void (Hero)
  감정: 고요한 경외, 미지에 대한 호기심
  무드: 깊은 어둠 → 수중 빛의 산란
  온도: 차가움 (deep cool)

Section 2 — Observation
  감정: 고요한 관찰, 일기를 읽는 친밀감
  무드: 해저에서 올려다본 수면의 빛
  온도: 차가움에서 미지근함으로 전환 (cool → neutral)

Section 3 — Data Archive
  감정: 정밀함에 대한 경탄, 설계된 것이라는 확신
  무드: 청사진 위의 파동, 과학적 정적
  온도: 중립적 차가움 (neutral cool)

Section 4 — The Maker
  감정: 인간적 신뢰, 만든 이에 대한 존경
  무드: 서재의 따뜻함, 필름 그레인의 인물
  온도: 중립에서 따뜻함으로 (neutral → warm)

Section 5 — Archive
  감정: 발견의 기쁨, 수집욕
  무드: 유물을 대면하는 박물관 전시실
  온도: 따뜻한 중립 (warm neutral)

Section 6 — Ocean Circle
  감정: 소속감, 초대받는 특별함
  무드: 밀봉된 편지를 여는 순간
  온도: 따뜻함 (warm)

Section 7 — For Professionals
  감정: 전문성, 절제된 신뢰
  무드: 깨끗한 서류 위의 만년필
  온도: 중립 (neutral)
```

### 에디토리얼 미니멀리즘 섹션별 표현

| 섹션 | 미니멀리즘 표현 방식 |
|------|---------------------|
| The Void | 텍스트 최소화, 영상과 여백만으로 구성 |
| Observation | 일기 형식, 한 문장과 넓은 마진 |
| Data Archive | 데이터를 차트가 아닌 기하학 예술로 표현 |
| The Maker | 인물 사진 하나 + 편지 하나. 그 이상 없음 |
| Archive | 제품 카드당 정보 최소화, 탭/클릭으로 심화 |
| Ocean Circle | 폼 필드 최소화, 이메일 하나면 충분 |
| For Professionals | 기능적이되 과잉 정보 배제 |

## 3. 타이포그래피 체계

### 서체

- **타이틀/헤드라인 (영문)**: Cormorant Garamond, weight 300 (Light)
- **본문/UI (한글)**: Noto Sans KR, weight 300 (Light) / 400 (Regular)
- **본문/UI (영문)**: Noto Sans KR에 포함된 Latin 글리프 사용 또는 Cormorant Garamond 본문용

### 섹션 타이틀 처리 규칙

모든 영문 섹션 타이틀은 **소문자 + 마침표**로 표기한다.

### 한글 본문 특이사항

- 한글 본문은 양쪽 정렬(justify) 사용하지 않는다. 좌측 정렬(left-align)만 사용.
- 한글 줄 간격(line-height)은 영문 대비 넉넉하게 1.75~1.8 유지.
- 한글과 영문이 섞일 때 Cormorant Garamond + Noto Sans KR 폰트 스택 순서로 지정.

## 4. 컬러 시스템 — Abyssal Amber (확정)

> 확정일: 2026-02-22 | 팔레트명: "Abyssal Amber"
> Hue 연속성 원칙: 웜 블랙(H:30°) → 그레이지(H:35°) → 골드(H:38°) — 전체 팔레트가 같은 Hue 패밀리(30°~38°) 안에서 채도와 명도만 달라진다.

### 컬러 무드 원칙

- 전체적으로 **저채도(desaturated)**, **깊은 톤(deep tone)** 기조
- 비비드하거나 과포화된 색상 사용 금지
- 배경은 순백(#fff)이 아닌 미세한 톤을 가진 그레이지 또는 웜 블랙
- 텍스트는 순흑(#000)이 아닌 깊이감 있는 웜 다크
- "심해 = 블루"로 직역하지 않는다. "깊이 = 어둠 = 시간의 색(앰버)"로 재해석

### 컬러 토큰

```css
:root {
  /* ── 다크 시스템 ── */
  --void-bg:           #0A0908;  /* Abyssal Black — 웜 블랙, H:30° S:4% L:3.5% */
  --void-text:         #F1EFEB;  /* 따뜻한 오프화이트 */
  --navy:              #0D0B09;  /* Data Archive 배경 */
  --circle-bg:         #0B0A08;  /* Ocean Circle 배경 */

  /* ── 라이트 시스템 ── */
  --sand:              #E8E5E1;  /* Soft Greige — H:35° S:11% L:90% */
  --sand-deep:         #DDDAD5;  /* 진한 그레이지 변형 */
  --warm-ivory:        #ECEAE6;  /* 밝은 그레이지 */
  --archive-bg:        #E8E5E1;  /* Archive 섹션 배경 */
  --pro-bg:            #ECEAE6;  /* For Professionals 배경 */

  /* ── 악센트 ── */
  --amber:             #CCAD7B;  /* 샴페인 골드 — H:38° S:42% L:64% */
  --amber-muted:       #b89868;  /* 절제된 앰버 */
  --gold-particle:     rgba(204, 173, 123, 0.12);  /* 파티클/미세 요소 */
  --gold-glow:         rgba(180, 155, 110, 0.06);  /* 앰비언트 광원 */

  /* ── 라이트 텍스트 ── */
  --earth:             #312E2A;  /* 밝은 배경 위 기본 텍스트 */
  /* 불투명도 변형: rgba(49, 46, 42, 0.80/0.55/0.40/0.30/0.25/0.15/0.08) */

  /* ── 다크 텍스트 (어두운 배경 위) ── */
  /* 기본: #F1EFEB */
  /* 불투명도 변형: rgba(241, 239, 235, 0.45/0.35/0.3/0.2/0.15) */
  /* CTA: rgba(204, 173, 123, 0.45) — 골드 CTA */

  /* ── 보더/구분선 ── */
  /* 밝은 배경: rgba(49, 46, 42, 0.10/0.08/0.06) */
  /* 어두운 배경: rgba(241, 239, 235, 0.12/0.08/0.06) */
}
```

### 섹션별 컬러 적용

| 섹션 | 배경 | 텍스트 | 악센트 | 무드 키워드 |
|------|------|--------|--------|------------|
| The Void | `#0A0908` 웜 블랙 | `#F1EFEB` 오프화이트 | 골드 `#CCAD7B` CTA + 파티클 | 심해의 어둠 |
| Observation | `#E8E5E1` 그레이지 | `#312E2A` 웜 다크 | — | 관찰 일기 |
| Data Archive | `#0D0B09` 웜 블랙 | `#F1EFEB` 오프화이트 | 골드 파동 SVG, 앰버 글로우 | 설계된 정밀함 |
| The Maker | `#ECEAE6` 밝은 그레이지 | `#312E2A` 웜 다크 | — | 서재, 편지 |
| Archive | `#E8E5E1` 그레이지 | `#312E2A` 웜 다크 | — | 박물관 전시실 |
| Ocean Circle | `#0B0A08` 웜 블랙 | `#F1EFEB` 오프화이트 | 골드 링 애니메이션 | 밀봉된 초대장 |
| For Professionals | `#ECEAE6` 밝은 그레이지 | `#312E2A` 웜 다크 | — | 깨끗한 서류 |

### 배경 컬러 프로그레션

```
Dark → Light → Dark → Light → Light → Dark → Light
(Void)  (Obs)  (Data)  (Maker) (Archive) (Circle) (Pro)
#0A0908 #E8E5E1 #0D0B09 #ECEAE6 #E8E5E1 #0B0A08 #ECEAE6
```

이 명암 교차가 스크롤 시 리듬감을 만든다. 어두운 섹션 → 밝은 섹션 → 어두운 섹션의 반복이 깊이 변화를 감각적으로 전달한다.

### 텍스트 대비 원칙

- 어두운 배경: `#F1EFEB` 오프화이트 (WCAG AA 이상 명도비 확보)
- 밝은 배경: `#312E2A` 웜 다크 (동일 기준)
- 라이트 텍스트에 순백(#fff) 사용 지양 — 따뜻한 오프화이트 유지
- 다크 텍스트에 순흑(#000) 사용 지양 — 웜 다크 브라운 유지

### 악센트 사용 철학

- 악센트 컬러는 **샴페인 골드 `#CCAD7B`** 하나만 사용한다
- 악센트는 CTA, 호버 상태, 파티클, 파동 SVG에만 제한적으로 사용
- 악센트가 "튀지" 않아야 한다. 낮은 불투명도(0.12~0.45)로 주변과 자연스럽게 어우러지되 시선을 끄는 정도
- 밝은 배경 위에서 골드 악센트 사용 시 불투명도를 더 낮춘다 (0.22~0.28)

## 5. 섹션 디바이더 — 한지 질감 + 수묵화 파동

### 비주얼 콘셉트

섹션 전환부에 한지(韓紙) 텍스처와 수묵화(水墨畫) 파동 패턴을 결합한 디바이더를 배치한다. 이 디바이더는 장식이 아니라 "깊이의 변화"를 감각적으로 전달하는 장치다.

### 한지 질감 처리

- 한지의 섬유질 텍스처를 배경에 매우 낮은 불투명도(opacity 3~8%)로 오버레이
- 밝은 배경 섹션: 미세한 밝은 한지 텍스처
- 어두운 배경 섹션: 미세한 어두운 한지 텍스처
- 텍스처는 "있는 듯 없는 듯" 수준. 의식하면 보이고, 의식하지 않으면 단색 배경으로 느껴지는 정도
- 구현: SVG 필터(feTurbulence + feDisplacementMap) 또는 실제 한지 스캔 이미지를 타일링

### 수묵화 파동 패턴

- 섹션 경계에 수묵화의 번짐 효과를 연상시키는 수평 파동 라인
- 높이: 80~120px
- 파동은 기하학적으로 단순화된 사인파 형태, 2~3개 레이어 중첩
- 불투명도: 각 레이어 5~15%
- 밝은 섹션 → 어두운 섹션 전환: 파동이 어두운 톤으로 번짐
- 어두운 섹션 → 밝은 섹션 전환: 파동이 밝은 톤으로 번짐
- 구현: SVG path + CSS animation (느린 수평 흐름, 주기 8~12초)

### 텍스처 통합 원칙

- 텍스처는 절대 콘텐츠를 방해하지 않는다
- 텍스처 위에 텍스트가 올라갈 경우 해당 영역의 텍스처 불투명도를 0%로 감소
- 디바이더 파동 애니메이션은 `prefers-reduced-motion: reduce` 시 정지 상태로 표시
- 모바일에서는 파동 레이어를 2개에서 1개로 줄여 성능 확보

### 디바이더 배치

```
[Section 1 — The Void]
  ─── 수묵 파동 디바이더 (dark → light) ───
[Section 2 — Observation]
  ─── 수묵 파동 디바이더 (light → dark) ───
[Section 3 — Data Archive]
  ─── 수묵 파동 디바이더 (dark → light) ───
[Section 4 — The Maker]
  ─── 한지 질감 페이드 (light → light, 미세한 톤 변화만) ───
[Section 5 — Archive]
  ─── 수묵 파동 디바이더 (light → dark) ───
[Section 6 — Ocean Circle]
  ─── 수묵 파동 디바이더 (dark → light) ───
[Section 7 — For Professionals]
```

Section 4→5 전환은 동일한 밝은 톤이므로 파동 없이 한지 질감의 미세한 톤 시프트만으로 구분한다.

## 7. 이미지 가이드

### 사진 처리 원칙

모든 사진은 아래 처리를 거쳐야 한다:

**필름 그레인**
- 오버레이 방식: CSS `background-image` 또는 SVG feTurbulence 필터
- 강도: 불투명도 10~20% (주제에 따라 조절)
- 입자 크기: 미세 (1~2px 단위)

**색보정 (Color Grading)**
- 채도: 원본 대비 -25~40% 감소
- 대비: 약간 낮춤 (하이라이트와 섀도우 차이 줄임)
- 톤 시프트: 미세한 따뜻한 톤 (섹션 무드에 따라 쿨/웜 조절)
- 블랙 포인트: 순수 블랙 사용하지 않음. 가장 어두운 부분도 약간의 톤 유지

**크롭 가이드라인**
- 피사체를 중앙에 놓지 않는다. 1/3 법칙 또는 비대칭 배치
- 넓은 여백(negative space)을 의도적으로 포함
- 수평선이 있는 사진: 상단 1/3 또는 하단 1/3에 배치

### 용도별 종횡비

| 용도 | Aspect Ratio | 비고 |
|------|-------------|------|
| 히어로/전폭 이미지 | 16:9 | 뷰포트 전체일 경우 동적 비율 |
| 인물 사진 | 3:4 (desktop), 4:5 (mobile) | |
| 제품 사진 | 3:4 | 세로형 강조 |
| 인라인 이미지 (본문 사이) | 16:9 또는 3:2 | 문맥에 따라 |
| 인증서/문서 | 7:10 | A4 비율 유사 |
| 썸네일 | 1:1 | 정방형 |

### 플레이스홀더 전략 (최종 에셋 확보 전)

최종 사진이 준비되기 전까지 아래 전략을 사용한다:

- **단색 블록**: 해당 섹션의 배경색과 유사하되 약간 다른 톤의 단색 영역
- **한지 텍스처**: 이미지 자리에 한지 텍스처를 대체 배치
- **그레이디언트**: 상단에서 하단으로 미세한 그레이디언트 (밝음 → 어둠 또는 반대)
- 절대 stock photo를 플레이스홀더로 사용하지 않는다
- 플레이스홀더에도 최종 이미지와 동일한 aspect-ratio를 적용한다
- `<img>` 태그에 `width`/`height` 속성 또는 CSS `aspect-ratio`를 반드시 지정하여 레이아웃 시프트 방지

## 8. 모션 원칙

### 전체 원칙

- 느리고 유기적이다. 빠르고 튀는 움직임은 럭셔리를 훼손한다.
- 물속에서의 움직임을 참조한다: 부력, 저항, 느린 관성.
- 모션은 콘텐츠를 보조하며, 모션 자체가 주인공이 되지 않는다.

### 이징 커브

| 이징 이름 | CSS 값 | 용도 | 이유 |
|----------|--------|------|------|
| gentle-in | cubic-bezier(0.25, 0.1, 0.25, 1.0) | 스크롤 진입 애니메이션 | 천천히 시작하여 자연스럽게 안착 |
| slow-fade | cubic-bezier(0.4, 0.0, 0.2, 1.0) | opacity 전환 | Material 기반이지만 더 느리게 적용 |
| water-ease | cubic-bezier(0.23, 1.0, 0.32, 1.0) | 호버, 인터랙션 반응 | 빠르게 반응하되 끝이 부드러운 수중 관성 |
| linear-slow | linear | 파동 애니메이션, 루프 | 일정한 속도의 끊김 없는 흐름 |

### 인터랙션 유형별 지속시간

| 인터랙션 유형 | 지속시간 | 이징 | 비고 |
|-------------|---------|------|------|
| 호버 반응 | 0.3~0.4s | water-ease | 즉각적이되 부드럽게 |
| 스크롤 진입 (fade-in) | 0.6~0.8s | gentle-in | 여유 있게 등장 |
| 스크롤 진입 (translate) | 0.6~0.8s | gentle-in | fade와 동시 적용 |
| 순차 진입 딜레이 | 0.1~0.2s per item | — | 항목 간 시차 |
| 페이지 전환 | 0.4~0.6s | slow-fade | 전체 화면 전환 시 |
| 파동 루프 | 8~15s per cycle | linear-slow | 끊김 없는 루프 |
| 데이터 카운트업 | 1.0~1.5s | easeOutQuart | 숫자 증가 애니메이션 |
| 버튼 상태 변화 | 0.3s | water-ease | 밑줄 두께, 불투명도 변화 |

### 스크롤 연동 vs 시간 기반

| 유형 | 적용 대상 | 구현 |
|------|----------|------|
| 스크롤 연동 (Scroll-linked) | 패럴랙스 (Observation 이미지), 압력 체험 효과, 파동 진폭 변화 | IntersectionObserver + scroll event (throttled) |
| 시간 기반 (Time-based) | 파동 루프 애니메이션, hover 반응, fade-in | CSS animation / transition |
| 스크롤 트리거 + 시간 기반 | 대부분의 스크롤 진입 애니메이션 (한 번 트리거되면 시간 기반으로 재생) | IntersectionObserver로 트리거, CSS animation으로 재생 |

### 애니메이션 금지 항목

다음은 절대 애니메이션을 적용하지 않는다:

- 텍스트 본문 (읽는 중 움직이면 가독성 훼손)
- 네비게이션 로고
- 폼 입력 중인 필드
- 인증서/QR 코드 (정보 전달 목적)
- 푸터 콘텐츠

### 접근성 대응

```css
@media (prefers-reduced-motion: reduce) {
  /* 모든 CSS animation 정지 */
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
  }

  /* 스크롤 연동 효과 비활성화 */
  /* JS에서 matchMedia 확인 후 패럴랙스/파동 정지 */
}
```

- `prefers-reduced-motion: reduce` 시 모든 스크롤 연동 애니메이션 비활성화
- 파동 애니메이션은 정지 상태의 정적 SVG로 대체
- fade-in 진입 효과는 즉시 표시(opacity: 1)로 대체
- 데이터 카운트업은 최종 값 즉시 표시

## 부록: 압력 체험 효과

마케팅 피드백에서 제안된 "스크롤 시 화면 선들이 눌리는 압력 체험 효과"에 대한 디자인 방향.

### 적용 범위

- Section 1(The Void)에서 Section 3(Data Archive)까지의 구간
- 이 구간은 깊이가 깊어지는 여정이므로 압력 체험이 의미를 가짐
- Section 4 이후는 수면 위로 올라오는 구간이므로 효과를 해제

### 비주얼 표현

- 텍스트나 이미지의 기하학적 변형은 적용하지 않는다 (가독성 보호)
- 대신 섹션 디바이더의 수묵화 파동이 스크롤에 따라 진폭이 미세하게 변화
- 배경 한지 텍스처의 scale이 스크롤 깊이에 따라 0.98~1.02 범위에서 미세하게 변화
- 전체적으로 "느껴지지만 의식하지 못하는" 수준의 미묘한 효과

### 기술 제약

- 성능 임팩트: CSS transform만 사용 (layout 재계산 유발하는 속성 금지)
- 60fps 유지 필수
- 모바일에서는 비활성화 (성능 우선)
- `will-change: transform` 사전 지정
